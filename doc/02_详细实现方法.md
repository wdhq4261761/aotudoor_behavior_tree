# AutoDoor 行为树系统 — 详细实现方法

## 1 行为树节点类型实现

### 1.1 节点基类 (Node)

所有节点的抽象基类，定义通用接口和默认行为：

```python
class Node(ABC):
    NODE_TYPE = "Node"                    # 类级别节点类型标识
    
    def __init__(self, node_id=None, config=None):
        self.node_id = node_id or uuid4()[:8]   # 唯一标识
        self.config = config or NodeConfig()     # 节点配置
        self.status = NodeStatus.SUCCESS         # 当前状态
        self.children: List[Node] = []           # 子节点列表
        self.parent: Optional[Node] = None       # 父节点引用
        self._tick_count = 0                     # tick计数
    
    @abstractmethod
    def tick(self, context: ExecutionContext) -> NodeStatus:
        """核心方法：执行节点逻辑，返回执行状态"""
        pass
```

**设计要点：**
- 使用 `uuid4()[:8]` 生成短 ID，兼顾唯一性与可读性
- `config` 使用 `NodeConfig` 数据类，支持 `extra` 字典存储节点特有参数
- `children` 列表维护子节点有序集合
- `tick()` 是纯抽象方法，子类必须实现

### 1.2 复合节点 (CompositeNode)

复合节点是行为树的核心控制结构，包含多个子节点并按特定策略执行。

#### 1.2.1 顺序节点 (SequenceNode)

**语义：** 按顺序依次执行子节点，所有子节点成功才返回成功，任一失败立即返回失败。

```
执行流程:
  child[0] ──SUCCESS──► child[1] ──SUCCESS──► child[2] ──SUCCESS──► return SUCCESS
     │                      │                      │
   FAILURE                FAILURE                FAILURE
     │                      │                      │
     ▼                      ▼                      ▼
  return FAILURE        return FAILURE         return FAILURE
```

**实现伪代码：**

```
function SequenceNode.tick(context):
    if children is empty:
        return SUCCESS
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING          // 等待当前子节点完成
        
        if status == FAILURE:
            current_index = 0       // 重置，下次从头开始
            return FAILURE          // 短路：任一失败即失败
        
        current_index += 1          // 成功则继续下一个
    
    current_index = 0               // 全部成功，重置
    return SUCCESS
```

**关键实现细节：**
- `current_index` 记录当前执行位置，支持 RUNNING 状态的记忆
- 失败时重置 `current_index = 0`，确保下次 tick 从头开始
- 跳过 `enabled = False` 的子节点

#### 1.2.2 选择节点 (SelectorNode)

**语义：** 按顺序执行子节点，任一成功即返回成功，全部失败才返回失败。

```
执行流程:
  child[0] ──FAILURE──► child[1] ──FAILURE──► child[2] ──FAILURE──► return FAILURE
     │                      │                      │
   SUCCESS                SUCCESS                SUCCESS
     │                      │                      │
     ▼                      ▼                      ▼
  return SUCCESS         return SUCCESS          return SUCCESS
```

**实现伪代码：**

```
function SelectorNode.tick(context):
    if children is empty:
        return FAILURE
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING
        
        if status == SUCCESS:
            current_index = 0       // 重置
            return SUCCESS          // 短路：任一成功即成功
        
        current_index += 1          // 失败则尝试下一个
    
    current_index = 0
    return FAILURE                  // 全部失败
```

#### 1.2.3 并行节点 (ParallelNode)

**语义：** 同时执行所有子节点，根据成功策略决定返回条件。

**成功策略：**
- `require_all`：所有启用的子节点成功才返回成功
- `require_one`：任一子节点成功即返回成功

**实现伪代码：**

```
function ParallelNode.tick(context):
    if children is empty:
        return SUCCESS
    
    success_count = 0
    failure_count = 0
    running_children = []
    
    for i, child in enumerate(children):
        if not child.config.enabled:
            continue
        
        // 检查缓存状态（已完成的子节点不再重复执行）
        if i in cached_statuses:
            status = cached_statuses[i]
            if status == SUCCESS: success_count += 1; continue
            if status == FAILURE: failure_count += 1; continue
        
        status = child.tick(context)
        
        if status == SUCCESS:
            cached_statuses[i] = SUCCESS
            success_count += 1
        elif status == FAILURE:
            cached_statuses[i] = FAILURE
            failure_count += 1
        elif status == RUNNING:
            running_children.append(child)
    
    // require_one 策略：任一成功即返回
    if success_policy == REQUIRE_ONE and success_count > 0:
        abort_running(context, running_children)
        return SUCCESS
    
    // 仍有运行中的子节点
    if running_children:
        return RUNNING
    
    // 所有子节点执行完毕
    if success_policy == REQUIRE_ALL:
        return SUCCESS if success_count == enabled_count else FAILURE
    else:
        return SUCCESS if success_count > 0 else FAILURE
```

**关键实现细节：**
- `cached_statuses` 缓存已完成子节点的状态，避免重复 tick
- `require_one` 模式下，一旦有子节点成功，立即中止其他运行中的子节点
- `reset()` 时清空缓存

### 1.3 条件节点 (ConditionNode)

条件节点检测环境状态，返回 SUCCESS 或 FAILURE，不产生副作用。

**基类实现：**

```python
class ConditionNode(Node):
    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self.invert = self.config.get("invert", False)           # 结果取反
        self.check_interval_ms = self.config.get("check_interval_ms", 300)  # 检测间隔
        self._last_check_time = 0
    
    def tick(self, context):
        // 检测间隔控制
        current_time = context.elapsed_time * 1000
        if current_time - self._last_check_time < self.check_interval_ms:
            return self.status    // 未到检测时间，返回上次状态
        
        self._last_check_time = current_time
        result = self._check_condition(context)   // 子类实现
        
        if self.invert:
            result = not result
        
        status = SUCCESS if result else FAILURE
        self.status = status
        
        // 条件成功时可选执行子节点
        if status == SUCCESS and self.children:
            return self._execute_children(context)
        
        return status
    
    @abstractmethod
    def _check_condition(self, context) -> bool:
        pass
```

**装饰参数（所有条件节点通用）：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| invert | 结果取反 | False |
| retry_count | 失败重试次数 | 0 |
| timeout_ms | 超时时间 | 0 |
| check_interval_ms | 检测间隔 | 300 |

#### 1.3.1 OCR检测节点 (OCRConditionNode)

```
执行流程:
  1. context.get_screenshot(region)     → 获取截图
  2. context.perform_ocr(image, keywords, language)  → OCR识别
  3. 若找到关键词:
     - blackboard.set(position_key, position)  → 记录位置（position_key 可配置）
     - return True
  4. 若未找到:
     - return False
```

**参数说明：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| region | 检测区域 | 全屏 |
| keywords | 检测关键词 | — |
| language | OCR语言 | "eng" |
| position_key | 位置保存的黑板变量名 | "last_detection_position" |

#### 1.3.2 图像匹配节点 (ImageConditionNode)

```
执行流程:
  1. Image.open(template_path)          → 加载模板图像（首次缓存）
  2. context.get_screenshot(region)     → 获取截图
  3. ImageProcessor.find_template(screenshot, template, threshold)  → 模板匹配
  4. 若匹配成功:
     - blackboard.set(position_key, position)  → 记录位置（position_key 可配置）
     - return True
  5. 若匹配失败:
     - return False
```

**参数说明：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| region | 检测区域 | 全屏 |
| template_path | 模板图像路径 | — |
| threshold | 匹配阈值 | 0.8 |
| position_key | 位置保存的黑板变量名 | "last_detection_position" |

#### 1.3.3 颜色检测节点 (ColorConditionNode)

```
执行流程:
  1. context.get_screenshot(region)     → 获取截图
  2. ImageProcessor.find_color(screenshot, target_color, tolerance)  → 颜色搜索
  3. 若找到匹配颜色:
     - blackboard.set(position_key, position)  → 记录位置（position_key 可配置）
     - return True
  4. 若未找到:
     - return False
```

**参数说明：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| region | 检测区域 | 全屏 |
| target_color | 目标颜色 | — |
| tolerance | 颜色容差 | 30 |
| position_key | 位置保存的黑板变量名 | "last_detection_position" |

#### 1.3.4 数字比较节点 (NumberConditionNode)

```
执行流程:
  1. context.get_screenshot(region)     → 获取截图
  2. OCRManager.recognize_number(screenshot, language)  → 数字OCR
  3. 若识别成功:
     - blackboard.set("last_number_value", value)
     - OPERATORS[operator](value, target_value)  → 比较运算
  4. 若识别失败:
     - return False
```

#### 1.3.5 变量判断节点 (VariableConditionNode)

```
执行流程:
  1. blackboard.get(variable_name)      → 获取变量值
  2. OPERATORS[operator](value, target_value)  → 比较运算
  3. 支持运算符: ==, !=, >, >=, <, <=, contains, exists, not_exists, not_contains
```

### 1.4 动作节点 (ActionNode)

动作节点执行具体操作，产生副作用。

**基类实现：**

```python
class ActionNode(Node):
    def tick(self, context):
        status = self._execute_action(context)   // 子类实现
        self.status = status
        
        // 动作成功后可选执行子节点
        if status == SUCCESS and self.children:
            return self._execute_children(context)
        
        return status
    
    @abstractmethod
    def _execute_action(self, context) -> NodeStatus:
        pass
```

**装饰参数（所有动作节点通用）：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| repeat_count | 重复次数 | 0 |
| timeout_ms | 超时时间 | 0 |

#### 各动作节点实现要点

| 节点 | 核心调用 | 关键参数 |
|------|----------|----------|
| KeyPressNode | `context.execute_key_press(key, action, duration)` | key, action(press/down/up), duration |
| MouseClickNode | `context.execute_mouse_click(button, position, action, duration)` | button, position, click_count |
| MouseMoveNode | `context.execute_mouse_move(position, relative)` | position, relative |
| DelayNode | `time.sleep()` + 状态记忆 | duration_ms, _start_time |
| SetVariableNode | `blackboard.set/increment/delete()` | variable_name, operation, value |
| ScriptNode | `ScriptExecutor.run_script()` | script_path, loop |
| CodeNode | `subprocess.run/Popen()` | code_path, wait_complete |
| AlarmNode | `AlarmPlayer.play()` | sound_path, volume, wait_complete |

**DelayNode 特殊实现：** DelayNode 是唯一可能返回 RUNNING 状态的动作节点。它使用 `_start_time` 记录开始时间，在 `duration_ms` 内持续返回 RUNNING，超时后返回 SUCCESS 并重置 `_start_time`。

## 2 节点间通信机制

### 2.1 黑板通信模式

节点间通过黑板系统进行解耦通信，遵循"写入-读取"模式：

```
┌──────────────────┐     set()      ┌──────────────┐     get()      ┌──────────────────┐
│  OCRConditionNode │ ─────────────► │              │ ◄───────────── │  MouseClickNode   │
│  (写入检测位置)    │                │   Blackboard │                │  (读取检测位置)    │
└──────────────────┘                │              │                └──────────────────┘
                                    │  ┌────────┐  │
┌──────────────────┐     set()      │  │变量字典 │  │     get()      ┌──────────────────┐
│  NumberCondition  │ ─────────────► │  │        │  │ ◄───────────── │  VariableCondition│
│  Node (写入数字)  │                │  │last_   │  │                │  Node (读取数字)  │
└──────────────────┘                │  │detection│ │                └──────────────────┘
                                    │  │_position││
                                    │  │last_   ││
                                    │  │number_ ││
                                    │  │value   ││
                                    │  │exec_   ││
                                    │  │count   ││
                                    │  └────────┘  │
                                    └──────────────┘
```

### 2.2 典型通信场景

**场景1：OCR检测 → 点击检测位置**

```
OCRConditionNode                    MouseClickNode
     │                                    │
     │ 1. _check_condition()              │
     │    screenshot = get_screenshot()   │
     │    found, pos = perform_ocr()      │
     │                                    │
     │ 2. blackboard.set(                 │
     │    "last_detection_position",      │
     │    pos)                            │
     │ ──────────────────────────────────►│
     │                                    │ 3. _execute_action()
     │                                    │    position = blackboard.get(
     │                                    │      "last_detection_position")
     │                                    │    execute_mouse_click(
     │                                    │      position=position)
```

**场景2：数字识别 → 条件判断**

```
NumberConditionNode                 VariableConditionNode
     │                                    │
     │ 1. _check_condition()              │
     │    value = ocr.recognize_number()  │
     │    blackboard.set(                 │
     │      "last_number_value", value)   │
     │ ──────────────────────────────────►│
     │                                    │ 2. _check_condition()
     │                                    │    value = blackboard.get(
     │                                    │      "last_number_value")
     │                                    │    compare(value, target)
```

### 2.3 观察者通知机制

黑板系统支持订阅变量变化，实现被动通知：

```python
# 订阅位置变化
def on_position_change(old_value, new_value):
    print(f"检测位置变化: {old_value} → {new_value}")

blackboard.subscribe("last_detection_position", on_position_change)

# 当 OCRConditionNode 设置位置时，自动触发回调
blackboard.set("last_detection_position", (100, 200))
# → on_position_change(None, (100, 200)) 被调用
```

### 2.4 执行上下文作为中介

ExecutionContext 是节点访问基础设施的唯一入口，实现了延迟初始化和统一接口：

```
Node.tick(context)
  │
  ├── context.get_screenshot(region)
  │     └── ScreenshotManager (延迟初始化)
  │           └── PIL.ImageGrab.grab()
  │
  ├── context.execute_key_press(key, action, duration)
  │     └── InputController (延迟初始化)
  │           └── PyAutoGUI / DD64.dll
  │
  ├── context.perform_ocr(image, keywords, language)
  │     └── OCRManager (延迟初始化)
  │           └── pytesseract → Tesseract
  │
  └── context.blackboard
        └── Blackboard (直接访问)
```

## 3 行为树序列化与反序列化

### 3.1 序列化方案

序列化采用"扁平化节点 + 连线列表"的结构，而非嵌套树结构，便于编辑器操作：

**序列化流程：**

```
Root Node (树结构)
    │
    │  collect_nodes() 递归遍历
    ▼
┌─────────────────────────────────────────┐
│ nodes: {                                │
│   "n1": { id, type, config, children }, │  ← 扁平化节点字典
│   "n2": { id, type, config, children }, │
│   "n3": { id, type, config, children }  │
│ }                                       │
│ connections: [                          │
│   { parent_id: "n1", child_id: "n2" }, │  ← 连线列表
│   { parent_id: "n1", child_id: "n3" }  │
│ ]                                       │
│ root_node: "n1"                         │  ← 根节点标识
└─────────────────────────────────────────┘
```

### 3.2 反序列化方案

**反序列化流程：**

```
JSON 数据
    │
    │ 1. 解析 nodes 字典
    ▼
┌──────────────────────────────────────┐
│ for each node_data in nodes:         │
│   node_class = NodeRegistry.get(type)│  ← 查找注册的节点类
│   node = node_class.from_dict(data)  │  ← 创建节点实例
│   nodes[node_id] = node              │
└──────────────────────────────────────┘
    │
    │ 2. 重建连接关系
    ▼
┌──────────────────────────────────────┐
│ for each conn in connections:        │
│   parent = nodes[conn.parent_id]     │
│   child = nodes[conn.child_id]       │
│   parent.add_child(child)            │  ← 重建父子关系
└──────────────────────────────────────┘
    │
    │ 3. 返回元组
    ▼
return (nodes[root_id], canvas_state, editor_state)
```

**返回值说明：**

`Serializer.deserialize()` 返回三元组：

| 元素 | 类型 | 说明 |
|------|------|------|
| root_node | Node | 反序列化的根节点 |
| canvas_state | Dict | 画布状态（节点位置、视口偏移、缩放比例） |
| editor_state | Dict | 编辑器状态（选中节点、文件路径） |

**引擎与编辑器的不同处理：**

```python
# 引擎加载：仅需根节点
result = Serializer.deserialize(data)
if isinstance(result, tuple):
    root_node = result[0]
else:
    root_node = result  # 兼容旧格式

# 编辑器加载：使用完整元组
root_node, canvas_state, editor_state = Serializer.deserialize(data)
self.set_tree_data(root_node, canvas_state, editor_state)
```

### 3.3 编辑器数据格式

编辑器使用扩展格式，包含画布状态信息：

```json
{
  "version": "2.0",
  "format_type": "behavior_tree_editor",
  "canvas": {
    "name": "未命名",
    "description": "",
    "viewport": {
      "zoom": 1.0,
      "offset_x": 0,
      "offset_y": 0
    }
  },
  "root_node": "node_1",
  "nodes": {
    "node_1": {
      "id": "node_1",
      "type": "SequenceNode",
      "name": "顺序",
      "enabled": true,
      "config": {},
      "position": { "x": 400, "y": 100 }
    }
  },
  "connections": [
    { "parent_id": "node_1", "child_id": "node_2" }
  ]
}
```

**与引擎格式的差异：**

| 特性 | 引擎格式 | 编辑器格式 |
|------|----------|------------|
| 节点位置 | 无 | `position: {x, y}` |
| 画布状态 | 无 | `canvas.viewport` |
| children 表示 | ID 列表 | 通过 connections 推导 |
| config 结构 | `NodeConfig.to_dict()` | 扁平化 key-value |

### 3.4 版本兼容

当前版本号 `CURRENT_VERSION = "2.0"`，序列化数据包含 `version` 字段用于兼容性检查。未来版本升级时，可在反序列化中添加版本迁移逻辑。

## 4 节点注册机制

### 4.1 注册流程

节点类型通过 `NodeRegistry` 进行集中注册，在模块导入时自动完成：

```python
# bt_nodes/actions/__init__.py
from .keyboard import KeyPressNode
from .mouse import MouseClickNode, MouseMoveNode
# ...

NodeRegistry.register("KeyPressNode", KeyPressNode)
NodeRegistry.register("MouseClickNode", MouseClickNode)
# ...
```

```python
# bt_nodes/conditions/__init__.py
from .ocr import OCRConditionNode
# ...

NodeRegistry.register("OCRConditionNode", OCRConditionNode)
# ...
```

### 4.2 核心节点注册

```python
# bt_core/registry.py
def register_core_nodes():
    NodeRegistry.register("SequenceNode", SequenceNode)
    NodeRegistry.register("SelectorNode", SelectorNode)
    NodeRegistry.register("ParallelNode", ParallelNode)
```

### 4.3 自定义节点扩展

要添加新的节点类型，需要：

1. 创建节点类，继承 `ActionNode` 或 `ConditionNode`
2. 实现 `tick()` / `_execute_action()` / `_check_condition()` 方法
3. 实现 `to_dict()` 和 `from_dict()` 方法
4. 在 `__init__.py` 中注册到 `NodeRegistry`
5. 在 `constants.py` 中添加显示名称和分类
6. 在 `property.py` 中添加配置 Schema

## 5 输入系统实现

### 5.1 双模式输入架构

```
ExecutionContext
    │
    ├── execute_key_press(key, action, duration)
    │     │
    │     └── InputController (标准版)
    │           ├── key_press() → pyautogui.press/keyDown/keyUp
    │           └── 支持 duration 参数（按住时长）
    │
    └── [DD版] AUTODOOR_USE_DD=1
          └── DDVirtualInput
                ├── key_down() → DD_key(dd_code, 1)
                ├── key_up() → DD_key(dd_code, 2)
                └── VK → DD码转换: DD_todc(vk_code)
```

## 6 画布编辑器实现

### 6.1 坐标系统

画布使用三层坐标变换：

```
世界坐标 (World)          ← 节点的逻辑位置
    │
    │ zoom + pan
    ▼
屏幕坐标 (Screen)         ← Canvas 上的像素位置
```

**变换公式：**

```
screen_x = world_x * zoom + pan_x
screen_y = world_y * zoom + pan_y

world_x = (screen_x - pan_x) / zoom
world_y = (screen_y - pan_y) / zoom
```

### 6.2 节点渲染

NodeItem 在 Tkinter Canvas 上绘制，包含以下视觉元素：

```
┌─────────────────────────┐
│ ○  │  节点名称    [✓]   │  ← 输入端口(上) + 颜色条 + 名称 + 状态指示器
└─────────────────────────┘
              ○               ← 输出端口(下)
```

| 元素 | Canvas Tag | 说明 |
|------|------------|------|
| shadow | node_shadow | 阴影效果 |

| color_bar | node_color | 左侧分类色条 |

| status_icon | node_icon | 状态图标 (✓/✗/⋯) |
| input_port | node_port_in | 输入端口圆 |
| output_port | node_port_out | 输出端口圆 |

### 6.3 连线渲染

连线使用贝塞尔曲线风格的三段折线：

```
parent output_port (start_x, start_y)
        │
        │ 垂直段
        ▼
   (start_x, mid_y) ────── (end_x, mid_y)    ← 水平段
                                │
                                │ 垂直段
                                ▼
              child input_port (end_x, end_y)
```

### 6.4 自动布局

当加载的树数据不包含位置信息时，使用递归自动布局：

```
function auto_layout(root_id):
    positions = {}
    
    function calc_subtree_width(node_id):
        children = get_children(node_id)
        if no children: return NODE_WIDTH (180)
        return max(sum(calc_subtree_width(c) for c in children), NODE_WIDTH)
    
    function layout_node(node_id, x, y):
        positions[node_id] = (x, y)
        children = get_children(node_id)
        if children:
            total_width = sum(calc_subtree_width(c) for c in children)
            current_x = x - total_width / 2
            for child in children:
                child_width = calc_subtree_width(child)
                layout_node(child, current_x + child_width/2, y + NODE_HEIGHT + V_GAP)
                current_x += child_width
    
    layout_node(root_id, 400, 100)
    return positions
```

## 10 快捷键实现细节

### 10.1 焦点保护机制

删除快捷键（Delete/Backspace）在输入框获得焦点时不触发节点删除：

```python
def _on_delete_key(self, event):
    """处理删除键，避免在输入框中误删节点"""
    focused = self.winfo_toplevel().focus_get()
    if focused:
        widget_type = str(type(focused).__name__)
        if widget_type in ("CTkEntry", "Entry", "CTkTextbox", "Text"):
            return None  # 允许输入框正常处理删除
    self._delete_selected()
    return "break"  # 阻止事件继续传播
```

### 10.2 快捷键绑定

**重要：快捷键必须绑定到顶级窗口**

快捷键绑定使用 `root.bind()` 而非 `self.bind()`，确保在画布获得焦点时快捷键仍能生效：

```python
def _bind_events(self):
    shortcuts = [
        ("<Control-z>", self.undo),
        ("<Control-y>", self.redo),
        ("<Control-Shift-Z>", self.redo),
        ("<Control-s>", lambda: self.save_tree()),
        ("<Control-Shift-S>", lambda: self.save_tree(save_as=True)),
        ("<Control-o>", lambda: self.load_tree()),
        ("<Control-n>", lambda: self.new_tree()),
        ("<Delete>", self._on_delete_key),
        ("<BackSpace>", self._on_delete_key),
        ("<Control-c>", self._copy_selected),
        ("<Control-v>", self._paste_selected),
        ("<Control-x>", self._cut_selected),
        ("<Control-d>", self._duplicate_selected),
    ]
    
    def make_handler(cb, key):
        def handler(e):
            if key in ("<Delete>", "<BackSpace>"):
                focused = self.winfo_toplevel().focus_get()
                if focused:
                    widget_type = str(type(focused).__name__)
                    if widget_type in ("CTkEntry", "Entry", "CTkTextbox", "Text"):
                        return None
            if callable(cb):
                cb()
            return "break"
        return handler
    
    root = self.winfo_toplevel()
    for key, callback in shortcuts:
        handler = make_handler(callback, key)
        root.bind(key, handler)
```

### 10.3 粘贴操作撤销支持

粘贴操作使用 `AddNodesCommand` 确保可撤销：

```python
def _paste_selected(self):
    if not self._clipboard_data:
        return
    
    # 创建节点副本并偏移位置
    new_nodes = []
    for node_data in self._clipboard_data:
        new_data = node_data.copy()
        new_data["position"] = {
            "x": node_data["position"]["x"] + 30,
            "y": node_data["position"]["y"] + 30
        }
        new_nodes.append(new_data)
    
    # 使用 CommandManager 支持撤销
    command = AddNodesCommand(self.canvas, new_nodes)
    self.command_manager.execute(command)
```

```python
def _start_auto_save(self):

def _auto_save(self):

        auto_save_path = self._get_auto_save_path()
        self._save_to_file(auto_save_path, include_editor_state=True)
```

### 11.2 崩溃恢复机制

```python
def _check_crash_recovery(self):
    """检查崩溃恢复，自动恢复无需确认"""
    auto_save_path = self._get_auto_save_path()
    if os.path.exists(auto_save_path):
        # 静默恢复，无需用户确认
        self.load_tree(auto_save_path)
        self._crash_recovered = True
```
