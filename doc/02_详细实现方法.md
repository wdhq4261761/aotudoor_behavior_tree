# AutoDoor 行为树系统 — 详细实现方法

## 1 行为树节点类型实现

### 1.1 节点基类 (Node)

所有节点的抽象基类，定义通用接口和默认行为：

```python
class Node(ABC):
    NODE_TYPE = "Node"                    # 类级别节点类型标识
    
    def __init__(self, node_id=None, config=None):
        self.node_id = node_id or uuid4()[:8]   # 唯一标识
        self.config = config or NodeConfig()     # 节点配置
        self.status = NodeStatus.SUCCESS         # 当前状态
        self.children: List[Node] = []           # 子节点列表
        self.parent: Optional[Node] = None       # 父节点引用
        self._tick_count = 0                     # tick计数
    
    @abstractmethod
    def tick(self, context: ExecutionContext) -> NodeStatus:
        """核心方法：执行节点逻辑，返回执行状态"""
        pass
```

**设计要点：**
- 使用 `uuid4()[:8]` 生成短 ID，兼顾唯一性与可读性
- `config` 使用 `NodeConfig` 数据类，采用扁平化结构存储节点参数（与原项目兼容）
- `children` 列表维护子节点有序集合
- `tick()` 是纯抽象方法，子类必须实现
- `reset(reset_counters=True)` 方法支持控制是否重置计数器，用于 retry/repeat 场景

**节点重置机制：**

节点重置是行为树状态管理的核心，需要区分不同场景：

| 场景 | 方法 | 行为 |
|------|------|------|
| 完全重置 | `reset(reset_counters=True)` | 重置所有状态和计数器 |
| 重试重置 | `_reset_for_retry()` | 设置 `status=RUNNING`，保留重试计数器 |
| 重复重置 | `_reset_for_repeat()` | 调用 `reset(False)`，保留重复计数器 |

**重要：** `_reset_for_retry()` 和 `_reset_for_repeat()` 必须将 `status` 设置为 `RUNNING`，确保节点在重试/重复期间返回正确状态。

### 1.2 复合节点 (CompositeNode)

复合节点是行为树的核心控制结构，包含多个子节点并按特定策略执行。

#### 1.2.1 顺序节点 (SequenceNode)

**语义：** 按顺序依次执行子节点，所有子节点成功才返回成功，任一失败立即返回失败。

```
执行流程:
  child[0] ──SUCCESS──► child[1] ──SUCCESS──► child[2] ──SUCCESS──► return SUCCESS
     │                      │                      │
   FAILURE                FAILURE                FAILURE
     │                      │                      │
     ▼                      ▼                      ▼
  return FAILURE        return FAILURE         return FAILURE
```

**实现伪代码：**

```
function SequenceNode.tick(context):
    if children is empty:
        return SUCCESS
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING          // 等待当前子节点完成
        
        if status == FAILURE:
            current_index = 0       // 重置，下次从头开始
            return FAILURE          // 短路：任一失败即失败
        
        current_index += 1          // 成功则继续下一个
    
    current_index = 0               // 全部成功，重置
    return SUCCESS
```

**关键实现细节：**
- `current_index` 记录当前执行位置，支持 RUNNING 状态的记忆
- 失败时重置 `current_index = 0`，确保下次 tick 从头开始
- 跳过 `enabled = False` 的子节点

#### 1.2.2 选择节点 (SelectorNode)

**语义：** 按顺序执行子节点，任一成功即返回成功，全部失败才返回失败。

```
执行流程:
  child[0] ──FAILURE──► child[1] ──FAILURE──► child[2] ──FAILURE──► return FAILURE
     │                      │                      │
   SUCCESS                SUCCESS                SUCCESS
     │                      │                      │
     ▼                      ▼                      ▼
  return SUCCESS         return SUCCESS          return SUCCESS
```

**实现伪代码：**

```
function SelectorNode.tick(context):
    if children is empty:
        return FAILURE
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING
        
        if status == SUCCESS:
            current_index = 0       // 重置
            return SUCCESS          // 短路：任一成功即成功
        
        current_index += 1          // 失败则尝试下一个
    
    current_index = 0
    return FAILURE                  // 全部失败
```

#### 1.2.3 并行节点 (ParallelNode)

**语义：** 同时执行所有子节点，根据成功策略决定返回条件。

**成功策略：**
- `require_all`：所有启用的子节点成功才返回成功
- `require_one`：任一子节点成功即返回成功

**实现伪代码：**

```
function ParallelNode.tick(context):
    if children is empty:
        return SUCCESS
    
    success_count = 0
    failure_count = 0
    running_children = []
    
    for i, child in enumerate(children):
        if not child.config.enabled:
            continue
        
        // 检查缓存状态（已完成的子节点不再重复执行）
        if i in cached_statuses:
            status = cached_statuses[i]
            if status == SUCCESS: success_count += 1; continue
            if status == FAILURE: failure_count += 1; continue
        
        status = child.tick(context)
        
        if status == SUCCESS:
            cached_statuses[i] = SUCCESS
            success_count += 1
        elif status == FAILURE:
            cached_statuses[i] = FAILURE
            failure_count += 1
        elif status == RUNNING:
            running_children.append(child)
    
    // require_one 策略：任一成功即返回
    if success_policy == REQUIRE_ONE and success_count > 0:
        abort_running(context, running_children)
        return SUCCESS
    
    // 仍有运行中的子节点
    if running_children:
        return RUNNING
    
    // 所有子节点执行完毕
    if success_policy == REQUIRE_ALL:
        return SUCCESS if success_count == enabled_count else FAILURE
    else:
        return SUCCESS if success_count > 0 else FAILURE
```

**关键实现细节：**
- `cached_statuses` 缓存已完成子节点的状态，避免重复 tick
- `require_one` 模式下，一旦有子节点成功，立即中止其他运行中的子节点
- `reset()` 时清空缓存

**日志输出实现：**

```python
def _tick_internal(self, context: ExecutionContext) -> NodeStatus:
    from bt_utils.log_manager import LogManager
    
    if not self.children:
        LogManager.instance().log_success(
            node_type="并行节点",
            node_name=self.name
        )
        return NodeStatus.SUCCESS
    
    # ... 执行逻辑 ...
    
    if success_count == enabled_count:
        LogManager.instance().log_success(
            node_type="并行节点",
            node_name=self.name
        )
        return NodeStatus.SUCCESS
    else:
        LogManager.instance().log_failure(
            node_type="并行节点",
            node_name=self.name,
            reason=f"成功 {success_count}/{enabled_count} 个子节点"
        )
        return NodeStatus.FAILURE
```

#### 1.2.4 开始节点 (StartNode)

**语义：** 作为行为树的根节点，顺序执行所有子节点，但与 SequenceNode 不同的是，子节点失败后会继续执行后续子节点。

**继承关系：** StartNode 继承自 CompositeNode，使用 `current_index` 实现顺序执行逻辑。

```
执行流程:
  child[0] ──SUCCESS/FAILURE──► child[1] ──SUCCESS/FAILURE──► child[2] ──► ...
     │                              │                              │
     └──────────────────────────────┴──────────────────────────────┘
                    所有子节点执行完毕后返回
     │
     ▼
  if has_failure: return FAILURE
  else: return SUCCESS
```

**与 SequenceNode 的区别：**

| 特性 | SequenceNode | StartNode |
|------|--------------|-----------|
| 失败处理 | 任一失败立即返回 | 失败后继续执行 |
| 执行模式 | 短路执行 | 全部执行 |
| 用途 | 条件序列 | 行为树根节点 |
| 状态记忆 | 使用 current_index | 使用 current_index |

**实现伪代码：**

```
function StartNode.tick(context):
    if children is empty:
        log_success(node_type="开始节点", node_name=self.name)
        return SUCCESS
    
    has_failure = false
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING    // 等待当前子节点完成
        
        if status == FAILURE:
            has_failure = true
        // SUCCESS: 继续执行下一个子节点
        
        current_index += 1
    
    // 所有子节点执行完毕，重置索引
    current_index = 0
    
    if has_failure:
        log_failure(
            node_type="开始节点",
            node_name=self.name,
            reason="部分子节点执行失败"
        )
        return FAILURE
    else:
        log_success(node_type="开始节点", node_name=self.name)
        return SUCCESS
```

**关键实现细节：**
- `current_index` 记录当前执行位置，支持 RUNNING 状态的记忆
- 子节点失败时设置 `has_failure = true`，但继续执行后续子节点
- 所有子节点执行完毕后重置 `current_index = 0`
- 跳过 `enabled = False` 的子节点

### 1.3 条件节点 (ConditionNode)

条件节点检测环境状态，返回 SUCCESS 或 FAILURE，不产生副作用。

**基类实现：**

```python
class ConditionNode(Node):
    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self.invert = self.config.get("invert", False)           # 结果取反
        self.check_interval_ms = self.config.get("check_interval_ms", 300)  # 检测间隔
        self._last_check_time = 0
        self._children_running = False  # 子节点执行状态追踪标志
    
    def tick(self, context):
        // 检查子节点是否正在执行
        if self._children_running and self.children:
            return self._execute_children(context)
        
        // 检测间隔控制
        current_time = context.elapsed_time * 1000
        if current_time - self._last_check_time < self.check_interval_ms:
            return self.status    // 未到检测时间，返回上次状态
        
        self._last_check_time = current_time
        result = self._check_condition(context)   // 子类实现
        
        if self.invert:
            result = not result
        
        status = SUCCESS if result else FAILURE
        self.status = status
        
        // 条件成功时执行子节点
        if status == SUCCESS and self.children:
            // 通知 GUI 节点状态为 SUCCESS
            context.notify_node_status(self.node_id, "success")
            self._children_running = True
            return self._execute_children(context)
        
        return status
    
    @abstractmethod
    def _check_condition(self, context) -> bool:
        pass
```

**子节点执行状态追踪：**

ConditionNode 使用 `_children_running` 标志追踪子节点的执行状态：

| 属性 | 说明 |
|------|------|
| `_children_running` | 布尔标志，标记子节点是否正在执行 |

**状态追踪流程：**

```
ConditionNode 执行流程:
┌─────────────────────────────────────────────────────────────────────┐
│  1. 首次 tick:                                                       │
│     - 检查 _children_running 标志                                    │
│     - 若为 False，执行条件检测                                        │
│     - 若条件成功且有子节点:                                           │
│       - 通知 GUI 节点状态为 SUCCESS                                   │
│       - 设置 _children_running = True                                │
│       - 开始执行子节点                                                │
│                                                                      │
│  2. 后续 tick:                                                       │
│     - 检查 _children_running 标志                                    │
│     - 若为 True，跳过条件检测，继续执行子节点                          │
│     - 子节点执行完成后，重置 _children_running = False                │
└─────────────────────────────────────────────────────────────────────┘
```

**装饰参数（所有条件节点通用）：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| invert | 结果取反 | False |
| retry_count | 失败重试次数 | 0 |
| timeout_ms | 超时时间 | 0 |
| check_interval_ms | 检测间隔 | 300 |

#### 1.3.1 OCR检测节点 (OCRConditionNode)

```
执行流程:
  1. context.get_screenshot(region)     → 获取截图
  2. context.perform_ocr(image, keywords, language)  → OCR识别
  3. 若找到关键词:
     - blackboard.set(position_key, position)  → 记录位置（position_key 可配置）
     - return True
  4. 若未找到:
     - 输出识别到的所有文本内容
     - return False
```

**参数说明：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| region | 检测区域 | 全屏 |
| keywords | 检测关键词 | — |
| language | OCR语言 | "eng" |
| position_key | 位置保存的黑板变量名 | "last_detection_position" |
| preprocess_mode | 图像预处理模式 | "none" |

**日志输出实现（含识别文本）：**

```python
def _check_condition(self, context: ExecutionContext) -> bool:
    from bt_utils.log_manager import LogManager
    
    # ... 获取截图 ...
    
    found, position, all_text = context.perform_ocr(
        screenshot, self.keywords, self.language, self.preprocess_mode
    )
    
    if found:
        if position:
            context.blackboard.set(self.position_key, position)
        LogManager.instance().log_success(

    else:

        LogManager.instance().log_failure(
            node_type="OCR检测节点",
            node_name=self.name,
            reason=reason
        )
        return False
```

#### 1.3.2 图像匹配节点 (ImageConditionNode)

```
执行流程:
  1. Image.open(template_path)          → 加载模板图像（首次缓存）
  2. context.get_screenshot(region)     → 获取截图
  3. ImageProcessor.find_template(screenshot, template, threshold)  → 模板匹配
     返回: (found: bool, position: Optional[Tuple], confidence: float)
  4. 若匹配成功:
     - blackboard.set(position_key, position)  → 记录位置（position_key 可配置）
     - return True
  5. 若匹配失败:
     - 输出置信度和阈值信息

|------|------|--------|
| region | 检测区域 | 全屏 |
| template_path | 模板图像路径 | — |
| threshold | 匹配阈值（支持百分比和小数两种格式） | 0.8 |
| position_key | 位置保存的黑板变量名 | "last_detection_position" |

**threshold 参数兼容性：**

### 1.4 动作节点 (ActionNode)

动作节点执行具体操作，产生副作用。

**基类实现：**

```python
class ActionNode(Node):
    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self._children_running = False  # 子节点执行状态追踪标志
    
    def tick(self, context):
        // 检查子节点是否正在执行
        if self._children_running and self.children:
            return self._execute_children(context)
        
        status = self._execute_action(context)   // 子类实现
        self.status = status
        
        // 动作成功后执行子节点
        if status == SUCCESS and self.children:
            // 通知 GUI 节点状态为 SUCCESS
            context.notify_node_status(self.node_id, "success")
            self._children_running = True
            return self._execute_children(context)
        
        return status
    
    @abstractmethod
    def _execute_action(self, context) -> NodeStatus:
        pass
```

**子节点执行状态追踪：**

ActionNode 使用 `_children_running` 标志追踪子节点的执行状态：

| 属性 | 说明 |
|------|------|
| `_children_running` | 布尔标志，标记子节点是否正在执行 |

**状态追踪流程：**

```
ActionNode 执行流程:
┌─────────────────────────────────────────────────────────────────────┐
│  1. 首次 tick:                                                       │
│     - 检查 _children_running 标志                                    │
│     - 若为 False，执行动作                                            │
│     - 若动作成功且有子节点:                                           │
│       - 通知 GUI 节点状态为 SUCCESS                                   │
│       - 设置 _children_running = True                                │
│       - 开始执行子节点                                                │
│                                                                      │
│  2. 后续 tick:                                                       │
│     - 检查 _children_running 标志                                    │
│     - 若为 True，跳过动作执行，继续执行子节点                          │
│     - 子节点执行完成后，重置 _children_running = False                │
└─────────────────────────────────────────────────────────────────────┘
```

**GUI 状态通知机制：**

当 ActionNode 或 ConditionNode 成功执行后需要执行子节点时，必须先通知 GUI 节点状态为 SUCCESS，确保界面正确显示节点状态：

```python
# 在执行子节点前通知 SUCCESS 状态
if status == NodeStatus.SUCCESS and self.children:
    context.notify_node_status(self.node_id, "success")
    self._children_running = True
    return self._execute_children(context)
```

**装饰参数（所有动作节点通用）：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| repeat_count | 重复次数 | 0 |
| timeout_ms | 超时时间 | 0 |

**成功日志简化原则：**

动作节点的成功日志仅输出节点类型和名称，不包含额外信息（如位置、操作参数等），保持简洁：

```python
# 成功日志示例（简化后）
LogManager.instance().log_success(
    node_type="鼠标移动节点",
    node_name="移动到按钮位置"
)

# 失败日志示例（包含详细原因）
LogManager.instance().log_failure(
    node_type="鼠标点击节点",
    node_name="点击确认按钮",
    reason="点击操作失败"
)
```

#### 各动作节点实现要点

| 节点 | 核心调用 | 关键参数 |
|------|----------|----------|
| KeyPressNode | `context.execute_key_press(key, action, duration)` | key, action(press/down/up), duration |
| MouseClickNode | `context.execute_mouse_click(button, position, action, duration)` | button, position, click_count |
| MouseMoveNode | `context.execute_mouse_move(position, relative, smooth)` | position, relative, smooth |
| MouseScrollNode | `context.execute_mouse_scroll(...)` | distance, clicks, direction |
| DelayNode | `time.sleep()` + 状态记忆 | duration_ms, _start_time |

| ScriptNode | `ScriptExecutor.run_script()` | script_path, loop |

**AlarmNode 实现细节：**

AlarmNode 是一个特殊的动作节点，用于播放报警音效：

```python
class AlarmNode(ActionNode):
    NODE_TYPE = "AlarmNode"

    def __init__(self, node_id: str = None, config: NodeConfig = None):
        super().__init__(node_id, config)
        
        from bt_utils.resource_manager import ResourceManager
        from config.settings_manager import SettingsManager
        
        # 从设置中获取默认值
        default_sound = ResourceManager().get_alarm_sound_path()
        default_volume = SettingsManager().get("alarm_volume", 70)
        
        self.sound_path = self.config.get("sound_path", default_sound)
        self.volume = self.config.get_int("volume", default_volume)
        self.wait_complete = self.config.get_bool("wait_complete", True)
        self._abort_flag = False

    def _execute_action(self, context) -> NodeStatus:
        from bt_utils.alarm import AlarmPlayer
        player = AlarmPlayer()

        # 解析相对路径为绝对路径
        resolved_sound_path = context.resolve_path(self.sound_path)
        player.play(resolved_sound_path, self.volume, self.wait_complete)
        
        return NodeStatus.SUCCESS

    def abort(self, context) -> None:
        """中止时立即停止播放"""
        self._abort_flag = True
        from bt_utils.alarm import AlarmPlayer
        AlarmPlayer().stop()
        super().abort(context)
```

**参数设计说明：**

| 参数 | 说明 | 来源 |
|------|------|------|
| sound_path | 音效文件路径 | 节点配置（支持相对路径 `./audio/alarms/...`） |
| volume | 音量 (0-100) | 节点配置，默认从设置读取 |
| wait_complete | 是否等待播放完成 | 节点配置 |
| repeat_count | 重复次数 | 装饰器参数（不在节点配置中） |
| repeat_interval_ms | 重复间隔 | 装饰器参数（不在节点配置中） |

**重要：** `repeat_count` 和 `repeat_interval_ms` 是装饰器参数，通过 `ACTION_DECORATOR_FIELDS` 定义，由 `_execute_with_decorators()` 处理重复逻辑。节点配置中不应包含这些参数，避免 GUI 显示重复字段。

**相对路径解析：**

AlarmNode 的 `sound_path` 支持相对路径格式（如 `./audio/alarms/alarm.mp3`），在执行时通过 `context.resolve_path()` 解析为绝对路径：

```python
# 相对路径
sound_path = "./audio/alarms/custom_alarm.mp3"

# 执行时解析
resolved_path = context.resolve_path(sound_path)
# → "D:/Projects/MyProject/audio/alarms/custom_alarm.mp3"
```

这确保了项目可以在不同环境中正确加载音频资源。

**DelayNode 特殊实现：** DelayNode 是唯一可能返回 RUNNING 状态的动作节点。它使用 `_start_time` 记录开始时间，在 `duration_ms` 内持续返回 RUNNING，超时后返回 SUCCESS 并重置 `_start_time`。

**MouseMoveNode 平滑移动实现：** MouseMoveNode 支持 `smooth` 参数，启用时使用 smoothstep 缓动函数实现平滑移动效果。平滑移动通过 InputController 的 `mouse_move` 方法实现，将移动过程分解为多个小步，每步使用 smoothstep 插值计算位置，使鼠标移动更加自然流畅。

**MouseScrollNode 滚动实现：** MouseScrollNode 支持 `distance`（滚动距离）、`clicks`（点击次数）、`direction`（滚动方向）三个参数，与旧项目保持兼容。

**ScriptNode 执行器池管理：**

ScriptNode 使用类级别的执行器池管理脚本执行状态，确保节点重置时正确清理：

```python
class ScriptNode(ActionNode):
    _executor_pool: Dict[str, ScriptExecutor] = {}  # 类级别执行器池
    _pool_lock = threading.Lock()  # 线程安全锁
    
    def reset(self, reset_counters: bool = True) -> None:
        with self._lock:
            # 停止当前执行器
            if self._executor and self._executor.is_running:
                self._executor.stop_script()
            
            # 从执行器池中移除
            with ScriptNode._pool_lock:
                if self.node_id in ScriptNode._executor_pool:
                    del ScriptNode._executor_pool[self.node_id]
            
            self._executor = None
            self._script_started = False
            self._aborted = False
        super().reset(reset_counters=reset_counters)
```

**ScriptNode 循环执行逻辑：**

ScriptNode 的 `loop` 参数与 `repeat_count` 参数需要正确配合：

```python
# loop=true 且 repeat_count=0 时使用循环执行
use_loop = self.loop and self.config.repeat_count == 0
self._executor.run_script(self._script_content, loop=use_loop)
```

当 `repeat_count != 0` 时，由装饰器 `_execute_with_decorators()` 处理重复逻辑，不应同时启用 `loop`。

**ScriptExecutor delay 命令非阻塞实现：**

脚本中的 `delay` 命令需要支持中断，不能使用阻塞式 `time.sleep()`：

```python
elif command["type"] == "delay":
    delay_time = command["time"] / 1000
    elapsed = 0
    # 分段检查 is_running，支持中断
    while elapsed < delay_time and self.is_running:
        sleep_time = min(0.1, delay_time - elapsed)
        time.sleep(sleep_time)
        elapsed += sleep_time
```

这确保了用户点击停止时，脚本能够及时响应并停止执行。

## 2 节点间通信机制

### 2.1 日志管理器 (LogManager)

LogManager 是节点日志输出的核心组件，采用单例模式，提供统一的日志接口：

```python
class LogManager:
    _instance = None
    
    @classmethod
    def instance(cls) -> "LogManager":
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance
    
    def log_success(self, node_type: str, node_name: str) -> None:
        """记录节点执行成功"""
        self._add_log("SUCCESS", node_type, node_name)
    
    def log_failure(self, node_type: str, node_name: str, reason: str = "") -> None:
        """记录节点执行失败"""
        message = f"{node_name}: 失败"
        if reason:
            message += f" - {reason}"
        self._add_log("FAILURE", node_type, message)
    
    def log_aborted(self, node_type: str, node_name: str) -> None:
        """记录节点被中止"""
        self._add_log("ABORTED", node_type, f"{node_name}: 已中止")
```

**日志级别说明：**

| 级别 | 颜色 | 用途 |
|------|------|------|
| SUCCESS | 绿色 | 节点执行成功 |
| FAILURE | 红色 | 节点执行失败，需提供原因 |
| ABORTED | 橙色 | 节点被用户中止 |

## 3 行为树序列化与反序列化

### 3.1 序列化方案

序列化采用"扁平化节点 + 连线列表"的结构，而非嵌套树结构，便于编辑器操作：

**序列化流程：**

```
Root Node (树结构)
    │
    │  collect_nodes() 递归遍历
    ▼
┌─────────────────────────────────────────┐
│ nodes: {                                │
│   "n1": { id, type, config, children }, │  ← 扁平化节点字典
│   "n2": { id, type, config, children }, │
│   "n3": { id, type, config, children }  │
│ }                                       │
│ connections: [                          │
│   { parent_id: "n1", child_id: "n2" }, │  ← 连线列表
│   { parent_id: "n1", child_id: "n3" }  │
│ ]                                       │
│ root_node: "n1"                         │  ← 根节点标识
└─────────────────────────────────────────┘
```

### 3.2 反序列化方案

**反序列化流程：**

```
JSON 数据
    │
    │ 1. 解析 nodes 字典
    ▼
┌──────────────────────────────────────┐
│ for each node_data in nodes:         │
│   node_class = NodeRegistry.get(type)│  ← 查找注册的节点类
│   node = node_class.from_dict(data)  │  ← 创建节点实例
│   nodes[node_id] = node              │
└──────────────────────────────────────┘
    │
    │ 2. 重建连接关系
    ▼
┌──────────────────────────────────────┐
│ for each conn in connections:        │
│   parent = nodes[conn.parent_id]     │
│   child = nodes[conn.child_id]       │
│   parent.add_child(child)            │  ← 重建父子关系
└──────────────────────────────────────┘
    │
    │ 3. 返回元组
    ▼
return (nodes[root_id], canvas_state, editor_state)
```

**返回值说明：**

`Serializer.deserialize()` 返回三元组：

| 元素 | 类型 | 说明 |
|------|------|------|
| root_node | Node | 反序列化的根节点 |
| canvas_state | Dict | 画布状态（节点位置、视口偏移、缩放比例） |
| editor_state | Dict | 编辑器状态（选中节点、文件路径） |

**引擎与编辑器的不同处理：**

```python
# 引擎加载：仅需根节点
result = Serializer.deserialize(data)
if isinstance(result, tuple):
    root_node = result[0]
else:
    root_node = result  # 兼容旧格式

# 编辑器加载：使用完整元组
root_node, canvas_state, editor_state = Serializer.deserialize(data)
self.set_tree_data(root_node, canvas_state, editor_state)
```

### 3.3 编辑器数据格式

编辑器使用扩展格式，包含画布状态信息：

```json
{
  "version": "2.0",
  "format_type": "behavior_tree_editor",
  "canvas": {
    "name": "未命名",
    "description": "",
    "viewport": {

      "offset_x": 0,
      "offset_y": 0
    }
  },
  "root_node": "node_1",
  "nodes": {
    "node_1": {
      "id": "node_1",

| 特性 | 引擎格式 | 编辑器格式 |

**NodeConfig 扁平化序列化：**

`NodeConfig.to_dict()` 方法采用扁平化输出，将所有参数直接输出到 config 顶层：

```python
def to_dict(self) -> Dict[str, Any]:
    """序列化为字典（扁平化格式，与原项目兼容）"""
    result = {}
    if self.name:
        result["name"] = self.name
    if self.description:
        result["description"] = self.description
    result["enabled"] = self.enabled
    if self.retry_count != 0:
        result["retry_count"] = self.retry_count
    if self.repeat_count != 0:
        result["repeat_count"] = self.repeat_count
    if self.repeat_interval_ms != 100:
        result["repeat_interval_ms"] = self.repeat_interval_ms
    if self.timeout_ms != 0:
        result["timeout_ms"] = self.timeout_ms
    # 将 extra 中的参数合并到顶层
    if self.extra:
        for key, value in self.extra.items():
            result[key] = value
    return result
```

**节点序列化示例：**

```python
# OCRConditionNode.to_dict() 示例
def to_dict(self) -> Dict[str, Any]:
    data = super().to_dict()
    # 直接写入 config 顶层，不使用 extra 嵌套
    data["config"]["region"] = self.region
    data["config"]["keywords"] = self.keywords
    data["config"]["language"] = self.language
    data["config"]["position_key"] = self.position_key
    return data
```

当前版本号 `CURRENT_VERSION = "2.0"`，序列化数据包含 `version` 字段用于兼容性检查。未来版本升级时，可在反序列化中添加版本迁移逻辑。

### 5.2 DD虚拟键盘实现细节

**DLL加载流程：**

```
1. 搜索路径优先级:
   a. 构造参数指定的 dll_path
   b. bt_utils/DD64.dll
   c. bt_utils/drivers/DD64.dll
   d. ../drivers/DD64.dll
   e. 项目根/drivers/DD64.dll

2. 加载验证:
   dd_dll = ctypes.windll.LoadLibrary(path)
   result = dd_dll.DD_btn(0)    // 测试调用
   if result == 1: available = True

3. 按键转换链:
   key_name → VK_CODE_MAP → vk_code → DD_todc(vk_code) → dd_code
   示例: "enter" → 0x0D → DD_todc(0x0D) → 28

function auto_layout(root_id):
    positions = {}
    
    function calc_subtree_width(node_id):
        children = get_children(node_id)
        if no children: return NODE_WIDTH (180)
        return max(sum(calc_subtree_width(c) for c in children), NODE_WIDTH)
    
    function layout_node(node_id, x, y):
        positions[node_id] = (x, y)
        children = get_children(node_id)
        if children:
            total_width = sum(calc_subtree_width(c) for c in children)
            current_x = x - total_width / 2
            for child in children:
                child_width = calc_subtree_width(child)
                layout_node(child, current_x + child_width/2, y + NODE_HEIGHT + V_GAP)
                current_x += child_width
    
    layout_node(root_id, 400, 100)
    return positions

        ("<Control-x>", self._cut_selected),
        ("<Control-d>", self._duplicate_selected),

                if focused:
                    widget_type = str(type(focused).__name__)
                    if widget_type in ("CTkEntry", "Entry", "CTkTextbox", "Text"):
                        return None
            if callable(cb):
                cb()
            return "break"
        return handler
    
    root = self.winfo_toplevel()
    for key, callback in shortcuts:
        handler = make_handler(callback, key)
        root.bind(key, handler)

## 12 项目工程文件管理系统

### 12.1 PathResolver 路径解析器

PathResolver 负责相对路径与绝对路径之间的转换，确保项目的可移植性。

```python
class PathResolver:
    """资源路径解析器"""
    
    def __init__(self, project_root: str):
        self.project_root = os.path.abspath(project_root)
    
    def to_relative(self, absolute_path: str) -> str:
        """将绝对路径转换为相对路径"""
        abs_path = os.path.abspath(absolute_path)
        rel_path = os.path.relpath(abs_path, self.project_root)
        return f"./{rel_path.replace(os.sep, '/')}"
    
    def to_absolute(self, relative_path: str) -> str:
        """将相对路径转换为绝对路径"""
        if relative_path.startswith("./"):
            relative_path = relative_path[2:]
        
        abs_path = os.path.join(self.project_root, relative_path)
        return os.path.normpath(abs_path)
    
    def is_valid_relative_path(self, path: str) -> bool:
        """检查相对路径是否有效"""
        if not path.startswith("./"):
            return False
        
        abs_path = self.to_absolute(path)
        if not os.path.exists(abs_path):
            return False
        
        real_abs_path = os.path.realpath(abs_path)
        real_project_root = os.path.realpath(self.project_root)
        
        return real_abs_path.startswith(real_project_root)
```

**设计要点：**
- 使用 `os.path.relpath()` 计算相对路径
- 统一使用 `/` 作为路径分隔符（跨平台兼容）
- 路径有效性验证防止目录遍历攻击

### 12.2 ResourceImporter 资源导入器

ResourceImporter 自动导入资源文件到项目目录并生成相对路径引用。

```python
class ResourceImporter:
    """资源导入管理器"""
    
    def __init__(self, project_root: str):
        self.project_root = project_root
        self.resolver = PathResolver(project_root)
    
    def import_resource(self, source_path: str, resource_type: str = None) -> str:
        """导入资源文件到项目目录"""
        if resource_type is None:
            resource_type = self._detect_resource_type(source_path)
        
        target_dir = self._get_target_directory(resource_type)
        filename = os.path.basename(source_path)
        filename = self._handle_name_conflict(target_dir, filename)
        
        target_path = os.path.join(self.project_root, target_dir, filename)
        
        os.makedirs(os.path.dirname(target_path), exist_ok=True)
        shutil.copy2(source_path, target_path)
        
        return self.resolver.to_relative(target_path)
    
    def _detect_resource_type(self, path: str) -> str:
        """检测资源类型"""
        ext = os.path.splitext(path)[1].lower()
        
        if ext in ['.png', '.jpg', '.jpeg', '.bmp', '.gif']:
            return 'image'
        elif ext in ['.py', '.bat', '.sh', '.ps1']:
            return 'script'
        elif ext in ['.mp3', '.wav', '.ogg', '.flac']:
            return 'audio'
        elif ext in ['.json', '.yaml', '.xml', '.txt']:
            return 'data'
        else:
            return 'other'
```

### 12.4 PackageExporter 打包导出器

PackageExporter 将项目打包为 ZIP 文件，便于分享和分发。

```python
class PackageExporter:
    """项目打包导出器"""
    
    EXCLUDE_DIRS = ['.metadata', '__pycache__', '.git', 'node_modules', 'cache']
    EXCLUDE_FILES = ['tree_backup_', 'screenshot_', '.DS_Store', 'Thumbs.db']
    EXCLUDE_EXTENSIONS = ['.pyc', '.pyo', '.tmp', '.bak', '.log']
    
    def __init__(self, project_root: str):
        self.project_root = project_root
    
    def export_to_zip(self, output_path: str = None) -> str:
        """导出项目为 ZIP 文件"""
        if output_path is None:
            project_name = os.path.basename(self.project_root)
            output_path = f"{project_name}.zip"
        
        with zipfile.ZipFile(output_path, 'w', zipfile.ZIP_DEFLATED) as zipf:
            for root, dirs, files in os.walk(self.project_root):
                # 排除临时目录
                dirs[:] = [d for d in dirs if d not in self.EXCLUDE_DIRS]
                
                for file in files:
                    # 排除临时文件
                    if self._should_exclude_file(file):
                        continue
                    
                    file_path = os.path.join(root, file)
                    arcname = os.path.relpath(file_path, os.path.dirname(self.project_root))
                    zipf.write(file_path, arcname)
        
        # 生成使用说明
        self._generate_readme(output_path)
        
        return output_path

### 12.5 PackageImporter 项目导入器

PackageImporter 从 ZIP 压缩包导入项目到默认工作区。

```python
class PackageImporter:
    """项目导入器"""
    
    REQUIRED_FILES = ["project.json", "tree.json"]
    
    def __init__(self):
        pass
    
    def validate_package(self, zip_path: str) -> Tuple[bool, str]:
        """验证 ZIP 文件是否为有效的项目压缩包
        
        Args:
            zip_path: ZIP 文件路径
            
        Returns:
            Tuple[bool, str]: (是否有效, 错误信息)
        """
        if not os.path.exists(zip_path):
            return False, "文件不存在"
        
        if not zipfile.is_zipfile(zip_path):
            return False, "不是有效的 ZIP 文件"
        
        try:
            with zipfile.ZipFile(zip_path, 'r') as zipf:
                namelist = zipf.namelist()
                
                if not namelist:
                    return False, "ZIP 文件为空"
                
                found_project_json = False
                found_tree_json = False
                
                for name in namelist:
                    basename = os.path.basename(name)
                    if basename == "project.json":
                        found_project_json = True
                    elif basename == "tree.json":
                        found_tree_json = True
                
                if not found_project_json:
                    return False, "缺少 project.json 文件"
                
                if not found_tree_json:
                    return False, "缺少 tree.json 文件"
                
                return True, ""
                
        except zipfile.BadZipFile:
            return False, "ZIP 文件已损坏"
        except Exception as e:
            return False, f"读取 ZIP 文件失败: {str(e)}"
    
    def get_project_name(self, zip_path: str) -> Optional[str]:
        """从 ZIP 文件中获取项目名称"""
        try:
            with zipfile.ZipFile(zip_path, 'r') as zipf:
                namelist = zipf.namelist()
                
                if not namelist:
                    return None
                
                # 从第一个条目获取项目文件夹名
                first_entry = namelist[0]
                if '/' in first_entry:
                    project_folder = first_entry.split('/')[0]
                    return project_folder
                
                # 尝试从 project.json 获取项目名
                for name in namelist:
                    basename = os.path.basename(name)
                    if basename == "project.json":
                        import json
                        with zipf.open(name) as f:
                            project_config = json.load(f)
                            project_info = project_config.get("project_info", {})
                            return project_info.get("name", None)
                
                return None
                
        except Exception:
            return None
    
    def import_from_zip(
        self, 
        zip_path: str, 
        target_dir: str,
        overwrite: bool = False,
        new_name: Optional[str] = None
    ) -> Tuple[bool, str, Optional[str]]:
        """从 ZIP 文件导入项目
        
        Args:
            zip_path: ZIP 文件路径
            target_dir: 目标目录
            overwrite: 是否覆盖已存在的项目
            new_name: 新项目名称（用于重命名）
            
        Returns:
            Tuple[bool, str, Optional[str]]: (是否成功, 错误信息, 导入后的项目路径)
        """
        is_valid, error_msg = self.validate_package(zip_path)
        if not is_valid:
            return False, error_msg, None
        
        project_name = new_name or self.get_project_name(zip_path)
        if not project_name:
            project_name = os.path.splitext(os.path.basename(zip_path))[0]
        
        project_root = os.path.join(target_dir, project_name)
        
        if os.path.exists(project_root) and not overwrite:
            return False, "PROJECT_EXISTS", project_root
        
        try:
            os.makedirs(target_dir, exist_ok=True)
            
            if os.path.exists(project_root):
                shutil.rmtree(project_root)
            
            with zipfile.ZipFile(zip_path, 'r') as zipf:
                zip_root = self.get_project_root_in_zip(zip_path)
                
                for member in zipf.namelist():
                    if zip_root:
                        if not member.startswith(zip_root + "/"):
                            continue
                        relative_path = member[len(zip_root) + 1:]
                    else:
                        relative_path = member
                    
                    if not relative_path or relative_path.endswith("/"):
                        continue
                    
                    target_path = os.path.join(project_root, relative_path)
                    os.makedirs(os.path.dirname(target_path), exist_ok=True)
                    
                    with zipf.open(member) as source, open(target_path, 'wb') as target:
                        target.write(source.read())
            
            return True, "", project_root
            
        except Exception as e:
            return False, f"解压失败: {str(e)}", None
    
    def generate_new_name(self, base_name: str, target_dir: str) -> str:
        """生成不冲突的新项目名称"""
        if not os.path.exists(os.path.join(target_dir, base_name)):
            return base_name
        
        counter = 1
        while True:
            new_name = f"{base_name}_{counter}"
            if not os.path.exists(os.path.join(target_dir, new_name)):
                return new_name
            counter += 1
```

**验证规则：**
- 检查文件是否存在
- 检查是否为有效的 ZIP 文件
- 检查 ZIP 内是否包含 `project.json`
- 检查 ZIP 内是否包含 `tree.json`

**冲突处理策略：**
| 策略 | 说明 |
|------|------|
| 覆盖 | 删除现有项目目录，重新解压 |
| 重命名 | 自动添加后缀（如 MyProject_1, MyProject_2） |
| 取消 | 终止导入操作 |

### 12.6 ResourceService 资源管理服务

ResourceService 提供统一的资源管理功能，包括收集外部资源、导入资源到项目、缓存管理等。

```python
class ResourceService:
    """资源管理服务"""
    
    RESOURCE_TYPE_MAP = {
        'template_path': 'image',
        'script_path': 'script',
        'code_path': 'code',
        'sound_path': 'audio',
    }
    
    TYPE_DIR_MAP = {
        'image': 'images/templates',
        'script': 'scripts/script',
        'code': 'scripts/code',
        'audio': 'audio/alarms',
        'data': 'data/config',
        'other': 'data/other',
    }
    
    CACHE_DIR = 'cache'
    
    def __init__(self, project_root: str):
        self.project_root = project_root
        self.resolver = PathResolver(project_root)
    
    def collect_external_resources(self, tree_data: Dict) -> List[Dict]:
        """收集行为树中引用的外部资源"""
        external_resources = []
        nodes = tree_data.get('nodes', {})
        
        for node_id, node_data in nodes.items():
            for path_key, resource_type in self.RESOURCE_TYPE_MAP.items():
                path = node_data.get('config', {}).get(path_key)
                if path and not path.startswith('./'):
                    external_resources.append({
                        'node_id': node_id,
                        'path_key': path_key,
                        'path': path,
                        'resource_type': resource_type
                    })
        
        return external_resources
    
    def import_resources_to_project(self, resources: List[Dict]) -> Dict[str, str]:
        """批量导入资源到项目目录"""
        path_mapping = {}
        
        for resource in resources:
            new_path = self.import_single_file_to_project(
                resource['path'],
                resource['resource_type']
            )
            if new_path:
                path_mapping[resource['node_id'] + '.' + resource['path_key']] = new_path
        
        return path_mapping
    
    def import_single_file_to_project(
        self, 
        source_path: str, 
        resource_type: str,
        old_path: str = None
    ) -> str:
        """导入单个文件到项目目录"""
        target_dir = self.TYPE_DIR_MAP.get(resource_type, 'data/other')
        filename = os.path.basename(source_path)
        target_path = os.path.join(self.project_root, target_dir, filename)
        
        # 处理文件名冲突
        if os.path.exists(target_path):
            name, ext = os.path.splitext(filename)
            counter = 2
            while os.path.exists(target_path):
                filename = f"{name}_{counter}{ext}"
                target_path = os.path.join(self.project_root, target_dir, filename)
                counter += 1
        
        # 复制文件
        os.makedirs(os.path.dirname(target_path), exist_ok=True)
        shutil.copy2(source_path, target_path)
        
        # 如果有旧文件，移动到缓存目录
        if old_path:
            self.move_to_cache(old_path)
        
        return self.resolver.to_relative(target_path)
    
    def move_to_cache(self, file_path: str) -> bool:
        """将文件移动到缓存目录"""
        if not file_path or file_path.startswith('./'):
            return False
        
        if not os.path.exists(file_path):
            return False
        
        cache_dir = os.path.join(self.project_root, self.CACHE_DIR)
        os.makedirs(cache_dir, exist_ok=True)
        
        filename = os.path.basename(file_path)
        name, ext = os.path.splitext(filename)
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        cache_filename = f"{name}_{timestamp}{ext}"
        cache_path = os.path.join(cache_dir, cache_filename)
        
        shutil.move(file_path, cache_path)
        return True
    
    def update_tree_paths(self, tree_data: Dict, path_mapping: Dict[str, str]) -> Dict:
        """更新行为树中的路径引用"""
        nodes = tree_data.get('nodes', {})
        
        for key, new_path in path_mapping.items():
            node_id, path_key = key.rsplit('.', 1)
            if node_id in nodes:
                if 'config' not in nodes[node_id]:
                    nodes[node_id]['config'] = {}
                nodes[node_id]['config'][path_key] = new_path
        
        return tree_data
```

                if hasattr(self, 'behavior_tree') and self.behavior_tree:
                    self.behavior_tree.load_tree(last_file)
                    self._update_window_title()  # 更新窗口标题
                    print(f"[OK] 已自动加载上次文件: {last_file}")
            except Exception as e:
                print(f"[WARN] 自动加载上次文件失败: {e}")
    
    def _update_window_title(self):
        """更新窗口标题，显示项目名称"""
        project_name = None
        if hasattr(self.behavior_tree, 'project_root') and self.behavior_tree.project_root:
            project_name = os.path.basename(self.behavior_tree.project_root)
        
        if project_name:
            try:
                from main import VERSION
                self.title(f"autodoor - 行为树 {VERSION} - {project_name}")
            except ImportError:
                self.title(f"autodoor - 行为树 - {project_name}")
        else:
            self.title(_get_app_title())
```

## 14 UI 更新分发器实现（防止假卡死）

### 14.1 问题背景

tkinter 的 GUI 操作必须在主线程执行，但行为树引擎在独立线程中运行。如果直接从后台线程调用 tkinter 的 `update()` 或 `event_generate()`，可能导致：

1. **界面假卡死**：点击"开始运行"后，界面看起来卡住，节点状态不更新
2. **更新延迟**：需要鼠标点击才能触发界面更新
3. **再次运行异常**：手动停止后再次运行，界面节点高亮和状态灯显示异常

### 14.2 核心解决方案

采用**定时轮询 + 后台守护**混合模式：

```python
# bt_utils/ui_dispatcher.py

class UIUpdateDispatcher:
    def __init__(self):
        self._task_queue: Queue = Queue()  # 线程安全队列
        self._widget = None
        self._polling_active = False
        self._polling_interval_ms = 30  # 30ms 轮询间隔
        self._node_status_cache: Dict[str, tuple] = {}  # 节点状态缓存
    
    def dispatch_node_status(self, node_id: str, status: str, callback: Callable = None):
        """从后台线程调用，分发节点状态更新"""
        if node_id:
            self._node_status_cache[node_id] = ({"node_id": node_id, "status": status}, callback)
        
        task = UpdateTask(UpdateType.NODE_STATUS, {"node_id": node_id, "status": status}, callback)
        self._task_queue.put(task)  # 线程安全
        # 注意：不调用 after(0, ...)，完全依赖轮询机制
```

### 14.3 轮询机制实现

**关键：轮询必须在编辑器初始化时启动，而不是运行时启动。**

### 14.4 轮询守护机制

当 `_widget` 为 None 时，使用后台线程继续尝试调度：

```python
# bt_utils/ui_dispatcher.py

def _poll(self):
    """主线程轮询"""
    if not self._polling_active:
        return
    
    widget = self._widget
    
    try:
        if widget is not None:
            pending = self._task_queue.qsize()
            if pending > 0:
                self._process_updates()
    except Exception:
        pass
    
    if self._polling_active:
        if self._widget is not None:
            # 正常情况：主线程调度下一次轮询
            self._widget.after(self._polling_interval_ms, self._poll)
        else:
            # 异常情况：后台线程守护，等待 _widget 可用
            threading.Timer(self._polling_interval_ms / 1000, self._poll_daemon).start()

def _poll_daemon(self):
    """后台线程轮询，用于在 _widget 为 None 时继续尝试"""
    if self._polling_active and self._widget is not None:
        # _widget 可用，切换回主线程轮询
        self._widget.after(0, self._poll)
    elif self._polling_active:
        # 继续等待
        threading.Timer(self._polling_interval_ms / 1000, self._poll_daemon).start()
```

### 14.5 防止假卡死的关键要点

| 问题 | 错误做法 | 正确做法 |
|------|---------|---------|
| 启动时假卡死 | 在 `_start_running()` 中启动轮询 | 在 `_init_ui_dispatcher()` 中启动轮询 |
| 手动停止后异常 | 停止时调用 `stop_polling()` | **不要停止轮询**，让它持续运行 |
| 轮询永久停止 | `_widget` 为 None 时直接返回 | 使用后台线程守护，等待 `_widget` 可用 |
| `after(0, ...)` 不可靠 | 从后台线程调用 `after(0, ...)` | 完全依赖定时轮询机制 |
