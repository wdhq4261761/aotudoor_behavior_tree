## 1. 快速入门

### 1.1 什么是行为树？

行为树是一种可视化的自动化流程设计工具。通过拖拽节点并连接它们，您可以创建复杂的自动化任务，例如：

- 自动检测屏幕上的文字或图像
- 自动点击按钮
- 自动执行重复性操作

### 1.2 创建第一个行为树

**步骤1：创建新文件**

- 点击工具栏的【新建】按钮
- 或按快捷键 `Ctrl + N`

**步骤2：添加节点**

- 从左侧节点面板点击需要的节点
- 节点会自动添加到画布中央

**步骤3：连接节点**

- 从父节点底部的圆点拖拽到子节点顶部的圆点
- 释放鼠标完成连接

**步骤4：配置属性**

- 点击选中节点
- 在右侧属性面板配置参数

**步骤5：保存运行**

- 点击【保存】按钮保存文件
- 点击【运行】按钮执行行为树

## 3. 节点操作

### 3.1 添加节点

1. 在左侧节点面板找到需要的节点
2. 点击节点按钮
3. 节点自动添加到画布中央

### 3.2 选择节点

| 操作             | 效果         |
| -------------- | ---------- |
| 单击节点           | 选中单个节点     |
| Ctrl + 单击      | 多选节点（追加选择） |
| Ctrl + 单击已选中节点 | 取消选择该节点    |
| 左键拖动空白区域       | 框选多个节点     |
| Ctrl + 左键拖动    | 追加框选节点     |
| 单击空白区域         | 取消所有选中     |
| 单击连线           | 选中连线       |

**框选操作说明**：

1. 在画布空白区域按住鼠标左键并拖动
2. 拖动过程中会显示虚线选择框
3. 释放鼠标后，框内的节点将被选中
4. 如果需要追加框选的节点，可以在拖动前按住Ctrl键

```
框选示意：

    ┌─────────────────┐
    │   ┌───┐       │
    │   │ A │       │
    │   └───┘       │
    │     ┌───┐     │
    │     │ B │     │
    │     └───┘     │
    │  ○─────────○  │ ← 选择框
    └─────────────────┘

### 3.3 移动节点

**单节点移动**：

1. 在节点非端口区域按住鼠标左键
2. 拖拽到目标位置
3. 释放鼠标完成移动

**多节点批量移动**：

1. 使用框选或Ctrl+点击选中多个节点
2. 在任一选中节点上按住鼠标左键
3. 拖拽到目标位置，所有选中节点同步移动
4. 释放鼠标完成移动

**提示**：

- 移动节点时，连接线会自动更新位置
- 多节点移动时，节点间的相对位置保持不变
- 支持撤销操作（Ctrl+Z）

### 3.4 删除节点

| 方法   | 操作                          |
| ---- | --------------------------- |
| 快捷键  | 选中后按 `Delete` 或 `Backspace` |
| 右键菜单 | 右键点击 → 选择"删除"               |
| 批量删除 | 选中多个节点后按 `Delete`           |

**批量删除说明**：

1. 使用框选或Ctrl+点击选中多个节点
2. 按 `Delete` 键或右键菜单删除
3. 删除节点时会自动删除相关的连接线
4. 支持撤销操作（Ctrl+Z）

### 3.5 复制粘贴

| 快捷键      | 功能           |
| -------- | ------------ |
| Ctrl + C | 复制选中节点（支持多选） |
| Ctrl + V | 粘贴节点         |

**复制内容说明**：

复制节点时会完整复制以下内容：

- 节点名称
- 所有配置参数（如检测区域、关键词、阈值等）
- 节点启用状态
- 装饰参数（如重试次数、结果取反等）

**多节点复制粘贴**：

1. 选中多个节点（框选或Ctrl+点击）
2. 按 `Ctrl + C` 复制
3. 按 `Ctrl + V` 粘贴
4. 粘贴后自动选中新节点，取消原节点选中
5. 节点间的相对位置保持不变
6. 节点间的连接线也会一并复制
7. 支持撤销操作（Ctrl+Z）

**提示**：粘贴的节点会自动偏移50像素，避免完全重叠

## 4. 连接操作

### 4.1 节点端口说明

每个节点有两个端口：

- **输入端口**：节点顶部的圆点，用于接收父节点的连接
- **输出端口**：节点底部的圆点，用于连接子节点

```
        ○ 输入端口
    ┌─────────┐
    │  节点    │
    └─────────┘
        ○ 输出端口
```

### 4.2 创建连接

1. 将鼠标移到父节点的**输出端口**（底部圆点）
2. 光标变为十字形状时，按住左键拖拽
3. 拖拽到子节点的**输入端口**（顶部圆点）
4. 释放鼠标完成连接

```
创建连接示意：

    ┌─────────┐
    │ 父节点   │
    └────┬────┘
         ○ 输出端口
         │
         │  拖拽连线
         │
         ○ 输入端口
    ┌────┴────┐
    │ 子节点   │
    └─────────┘
```

### 4.3 删除连接

| 方法   | 操作              |
| ---- | --------------- |
| 快捷键  | 选中连线后按 `Delete` |
| 右键菜单 | 右键点击连线 → 选择"删除" |

### 4.4 连接规则

- 一个输出端口可以连接多个子节点
- 一个输入端口只能连接一个父节点
- 不能创建循环连接（子节点连接回父节点）

### 4.5 执行顺序

当组合节点有多个子节点时，连接线上会显示序号（1、2、3...）表示执行顺序。

**调整执行顺序**：

1. 删除需要调整的连接线
2. 按期望顺序重新连接

## 7. 节点类型详解

### 7.1 组合节点

#### 顺序节点

**用途**：按顺序执行所有子节点，全部成功才成功。

**执行流程**：

```
          顺序
           │
   ┌───────┼────────┐
   ↓       ↓        ↓
子节点1 → 子节点2 → 子节点3 → ...
  ↓失败       ↓失败       ↓失败
返回失败    返回失败    返回失败
```

**参数**：

| 参数      | 说明                                |
| ------- | --------------------------------- |
| 子节点间隔   | 每个子节点执行完成后的等待时间（毫秒）               |
| 失败后继续执行 | 开启后，中间节点失败仍继续执行后续节点，最终根据是否有失败判定结果 |

**失败后继续执行说明**：

默认情况下，顺序节点采用"短路失败"策略：任一子节点失败立即终止，后续节点不执行。

开启"失败后继续执行"后：

- 子节点失败时不会立即终止
- 继续执行后续所有子节点
- 所有子节点执行完毕后，检查是否有失败的节点
- 如果存在失败的子节点，顺序节点返回失败
- 如果所有子节点都成功，顺序节点返回成功

```
示例：顺序节点 [A] → [B] → [C] → [D]，开启"失败后继续执行"

情况：节点 B 执行失败

执行结果：
├── [A] 执行成功 ✓
├── [B] 执行失败 ✗ ← 记录失败，继续执行
├── [C] 执行成功 ✓ ← 仍然执行
└── [D] 执行成功 ✓ ← 仍然执行

最终结果：返回失败（因为 B 失败了）
```

**使用场景**：

- 需要按固定顺序执行的任务
- 流程化的自动化操作
- 需要执行所有清理操作，即使中间步骤失败
- 批量操作，需要知道每个操作的结果

**示例**：登录流程

```
顺序节点
├── OCR检测：检测登录按钮
├── 鼠标点击：点击登录按钮
├── 延时：等待2秒
└── OCR检测：检测登录成功
```

#### 选择节点

**用途**：按顺序执行子节点，任一成功即成功。

**执行流程**：

```
子节点1 → 子节点2 → 子节点3 → ...
  ↓成功       ↓成功       ↓成功
返回成功    返回成功    返回成功
```

**参数**：

- 子节点间隔：每个子节点执行完成后的等待时间（毫秒）

**使用场景**：

- 多种方案尝试，任一成功即可
- 容错处理，提供备选方案

**示例**：多方式登录

**获取模板图像的方式**：

1. **截图功能**（推荐）：

**截图功能说明**：

- 文件名格式：`template_YYYYMMDD_HHMMSS.png`（包含时间戳，避免重名）

**使用技巧**：

- 阈值越高，匹配越精确
- 建议阈值设置在80\~90之间

- 使用截图功能可以直接从屏幕获取模板，方便快捷

**参数**：

| 参数    | 说明                    |
| ----- | --------------------- |
| 检测区域  | 屏幕上的检测范围              |

| 提取模式  | 数字提取方式                |
| 比较模式  | 比较运算符                 |
| 比较值   | 用于比较的阈值               |
| 置信度阈值 | 识别置信度最低要求（0-100，默认50） |
| 位置变量名 | 保存检测位置的变量名（用于点击最近检测点） |

**预处理模式**：

| 模式   | 适用场景                   |
| ---- | ---------------------- |
| 普通文本 | 普通文本数字（默认）             |

**提取模式**：

| 模式  | 说明        | 示例              |
| --- | --------- | --------------- |
| 无规则 | 直接采集第一个数字 | "HP: 100" → 100 |

| 自定义 | 使用通配符自定义  | 见下方说明           |

**自定义提取模式**：

- 使用 `*` 表示要提取的数字
- 示例：
  - `"HP: *"` → 匹配 "HP: 100" 提取 100
  - `"(*/*)"` → 匹配 "(50/100)" 提取 50

**使用技巧**：

- 识别普通数字使用"普通文本"模式
- 识别游戏中的粗体、彩色、带特效的数字使用"艺术字"模式
- 如果识别不准确，可尝试降低置信度阈值
- 配合鼠标点击节点的"点击最近检测点"功能，可自动点击识别到的数字位置

**支持的运算符**：

| 运算符           | 说明          |

| !=            | 不等于         |
| >             | 大于          |
| <             | 小于          |
| >=            | 大于等于        |
| <=            | 小于等于        |
| exists        | 变量存在        |
| not\_exists   | 变量不存在       |
| contains      | 包含（字符串/列表）  |
| not\_contains | 不包含（字符串/列表） |

**类型自动推断**：

比较值会根据变量的实际类型自动转换，无需手动指定类型：

| 变量类型 | 转换规则                                             |
| ---- | ------------------------------------------------ |
| 数字   | 字符串比较值自动转换为数字                                    |
| 布尔值  | "true"/"1"/"yes" → True，"false"/"0"/"no" → False |
| 字符串  | 保持原样                                             |

**使用技巧**：

- 配合计数器实现循环控制
- 配合设变量节点实现状态机
- 使用 `exists` 判断变量是否已设置
- 使用 `contains` 判断字符串或列表是否包含某元素

**示例**：

```
计数器示例：

4. 变量判断：判断count>=10

类型推断示例：
变量 count = 10 (数字类型)
比较值 = "5" (字符串)
结果：自动将 "5" 转换为 5，执行 10 > 5 → 成功
```

- 自定义成功/失败音效

**使用技巧**：

- 自定义音效：选择自己的音频文件
- 多次提醒：设置重复次数>1，配合播放间隔
- 无限循环：设置重复次数=-1，持续播放直到手动停止
- 后台提示：取消勾选"等待播放完成"，不阻塞流程

**示例**：

```

OCR检测"错误" → 报警节点（默认设置）→ 点击"确定"按钮

示例2：自定义报警

## 8. 运行与调试

### 8.1 运行行为树

**启动运行**：

- 点击工具栏【运行】按钮（绿色）
- 或按快捷键 `空格键`

**运行状态显示**：

节点状态灯会在运行时自动显示，非运行时自动隐藏，避免遮挡节点名称。

| 状态  | 颜色   | 说明          |
| --- | ---- | ----------- |
| 空闲  | 默认   | 未执行（状态灯隐藏）  |
| 运行中 | 橙色闪烁 | 正在执行（状态灯显示） |
| 成功  | 绿色   | 执行成功（状态灯显示） |
| 失败  | 红色   | 执行失败（状态灯显示） |
| 已中止 | 灰色   | 被停止（状态灯显示）  |

**状态灯显示规则**：

- **非运行状态**：所有节点的状态灯自动隐藏，节点名称完整显示
- **运行状态**：状态灯自动显示，实时反映节点执行状态
- **停止运行**：所有节点状态重置，状态灯自动隐藏

### 8.2 停止执行

**停止方法**：

- 点击工具栏【停止】按钮（红色）
- 或按快捷键 `Escape`

### 8.3 查看日志

运行日志会显示在主界面的日志区域，包含：

- 节点执行状态
- 检测结果
- 错误信息

**日志级别**：

- `[INFO]`：普通信息
- `[BT]`：行为树相关
- `[ERROR]`：错误信息

### 8.4 调试技巧

1. **逐步测试**：先测试单个节点，再组合测试
2. **查看日志**：关注日志中的错误信息
3. **检查区域**：确保检测区域正确
4. **调整阈值**：图像匹配失败时尝试降低阈值
5. **使用延时**：在操作之间添加适当延时

### 9.5 项目管理（V3.1.0 新增）

**项目文件夹结构**：

每个行为树项目都是一个独立的文件夹，包含完整的行为树定义和所有关联资源：

```
MyProject/                    # 项目根文件夹
├── project.json              # 项目配置文件
├── tree.json                 # 行为树定义文件
├── tree_backup_*.json        # 自动备份文件（最多5个）
│
├── images/                   # 图像资源
│   ├── templates/            # 模板图片
│   └── screenshots/          # 截图文件
│
├── scripts/                  # 脚本资源
│   ├── python/               # Python脚本
│   └── batch/                # 批处理脚本
│
├── audio/                    # 音频资源
│   └── alarms/               # 报警音效
│
├── data/                     # 数据资源
│   └── config/               # 配置文件
│
└── docs/                     # 文档文件夹
```

**资源导入**：

1. 在节点属性面板中选择资源文件（图像、脚本、音频等）
2. 系统自动检测资源类型
3. 文件自动复制到项目对应目录
4. 自动生成相对路径引用（如 `./images/templates/button.png`）
5. 如遇文件名冲突，自动添加序号避免覆盖

**创建新项目**：

1. 点击工具栏【新建项目】按钮
2. 在弹出的对话框中填写：
   - 项目名称：用于标识项目
   - 保存位置：选择项目文件夹的父目录
   - 项目描述：可选的项目说明
3. 点击【创建】按钮
4. 系统自动创建项目文件夹结构
5. 新项目将自动打开，可以开始编辑行为树

**打开现有项目**：

1. 点击工具栏【打开项目】按钮
2. 在文件选择对话框中选择项目文件夹
3. 系统自动验证项目结构
4. 加载项目配置和行为树定义

**转换旧脚本为项目**：

如果您有旧的行为树脚本（.json文件），可以通过以下方式转换为项目：

**方式一：导出时自动转换**

1. 打开旧脚本文件（使用【打开】按钮）
2. 点击【导出】按钮
3. 系统检测到脚本不在项目文件夹中，会提示转换
4. 点击【是】，在弹出的对话框中设置项目信息
5. 系统自动创建项目文件夹并保存当前行为树

**方式二：手动创建项目**

1. 点击【新建项目】创建一个新项目
2. 使用【打开】按钮打开旧脚本文件
3. 全选旧脚本中的节点（Ctrl+A）
4. 复制节点（Ctrl+C）
5. 切换到新项目标签页
6. 粘贴节点（Ctrl+V）
7. 保存新项目

**项目导出**：

1. 点击工具栏【导出】按钮
2. 选择 ZIP 文件保存位置
3. 系统自动打包项目文件夹
4. 生成使用说明文档
5. 自动排除临时文件和缓存文件

**注意**：导出功能仅对项目文件夹有效。如果当前脚本不在项目文件夹中，系统会提示您先转换为项目。

**分享项目**：

1. 导出项目为 ZIP 文件
2. 将 ZIP 文件分享给其他用户
3. 接收者解压 ZIP 文件
4. 在编辑器中打开项目文件夹即可使用

**相对路径引用**：

所有资源文件使用相对路径引用，确保项目的可移植性：

```json
{
  "type": "ImageConditionNode",
  "config": {
    "template_path": "./images/templates/button.png",
    "threshold": 0.8
  }
}
```

**自动备份**：

- 每次保存项目时自动创建备份
- 备份文件命名：`tree_backup_YYYYMMDD_HHMMSS.json`
- 自动保留最近 5 个备份文件
- 旧备份自动清理

## 11. 常见问题

### Q1: 如何实现循环执行？

**A**: 设置组合节点的"重复次数"为 -1 即可实现无限循环。

### Q2: 点击位置不正确怎么办？

**A**:

1. 检查检测区域是否正确
2. 确保勾选了"点击最近检测点"
3. 检测节点会保存绝对坐标，点击节点会使用该坐标

### Q3: OCR识别不准确怎么办？

**A**:

1. 确保检测区域只包含需要识别的文字
2. 尝试更换OCR语言设置
3. 中文识别（简体/繁体）已针对中文特点优化，识别准确率显著提升
4. 系统会自动对图像进行预处理：
   - 中文：图像放大→中值滤波→灰度化→对比度增强2.5x→双锐化→二值化(阈值130)
   - 其他语言：灰度化→对比度增强1.5x→锐化→二值化(阈值128)

### Q4: 图像匹配失败怎么办？

**A**:

1. 降低匹配阈值（如从90降到80）
2. 确保模板图像清晰
3. 检查检测区域是否正确

### Q5: 如何使用"点击最近检测点"？

**A**:

1. 在条件节点（OCR/图像/颜色）中检测目标
2. 检测成功后位置会自动保存
3. 在点击节点中勾选"点击最近检测点"
4. 点击时会使用最近一次检测到的位置

### Q6: 修改属性后运行不生效怎么办？

**A**:

- 点击画布其他位置或切换选中节点，属性会自动保存
- 或手动按 `Ctrl + S` 保存

### Q7: 如何实现长按操作？

**A**:
在点击节点中设置：

- 动作：press
- 按住时长：500（毫秒）

### Q8: 如何实现拖拽操作？

**A**:
使用鼠标移动节点的拖拽功能：

1. 设置移动类型=拖拽
2. 设置位置为拖拽起点（或勾选"移动到最近检测点"）
3. 设置拖拽终点为目标位置
4. 设置拖拽时长（建议500-1000毫秒）

示例：

```
拖拽文件：
1. OCR检测：检测文件图标
2. 鼠标移动：移动类型=拖拽，移动到最近检测点=是，拖拽终点=(500,300)，拖拽时长=500
```

### Q9: 如何实现无限点击？

**A**:
在点击节点中设置：

- 点击次数：-1
- 点击间隔：100（毫秒）
- 节点会持续点击，直到按Escape键停止

使用场景：

- 需要持续点击直到满足条件
- 游戏中的连击操作
- 手动控制停止时机

### Q10: 节点执行顺序如何确定？

**A**:

- 连接线上会显示序号（1、2、3...）
- 按序号顺序执行
- 如需调整，删除连接线后重新按期望顺序连接

### Q10: 节点执行顺序如何确定？

**A**:

- 连接线上会显示序号（1、2、3...）
- 按序号顺序执行
- 如需调整，删除连接线后重新按期望顺序连接

### Q11: 如何提取特定格式的数字？

**A**:
使用数字比较节点的"自定义提取模式"：

- 要提取 "HP: 100/MAX" 中的 100
- 设置提取模式为"自定义"
- 设置自定义模式为 `HP: */MAX`
- 使用 `*` 表示要提取的数字部分
