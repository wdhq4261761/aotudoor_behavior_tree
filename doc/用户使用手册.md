## 1. 产品概述

### 1.1 什么是行为树？

行为树是一种可视化的自动化流程设计工具，通过图形化界面拖拽节点并连接它们，可以创建复杂的自动化任务。行为树广泛应用于游戏AI、RPA自动化、测试自动化等领域。

### 1.2 产品定位

AutoDoor 行为树系统是一个独立的可视化行为树编辑与执行框架，面向 Windows 平台的自动化场景，包括：

- **游戏辅助**：自动任务、自动战斗、自动采集等
- **RPA 流程**：自动化办公、数据录入、报表生成等
- **测试自动化**：UI测试、回归测试、压力测试等

### 1.3 核心特性

| 特性       | 说明                                     |
| -------- | -------------------------------------- |
| 可视化编辑器   | 基于 CustomTkinter 的节点式编辑器，支持拖拽、连线、缩放、框选 |
| 行为树引擎    | 独立线程执行，支持启动/暂停/停止/恢复                   |
| 17 种内置节点 | 3 种复合节点 + 5 种条件节点 + 9 种动作节点            |
| 黑板系统     | 观察者模式的数据共享机制，节点间解耦通信                   |
| 撤销/重做    | Command 模式，支持 100 步历史                  |
| DD 虚拟键盘  | 游戏级硬件模拟输入，绕过大多数输入检测                    |
| OCR 集成   | 内嵌 RapidOCR，支持中英文识别                    |
| 脚本录制     | TXT 脚本录制与回放                            |
| 项目管理     | 完整的项目文件管理系统，支持导入导出                     |

### 1.4 版本说明

系统提供两种打包版本：

| 版本  | 输入方式      | 适用场景          |
| --- | --------- | ------------- |
| 标准版 | PyAutoGUI | 普通应用自动化       |
| 游戏版 | DD64.dll  | 游戏自动化（绕过输入检测） |

## 2. 快速入门

### 2.1 创建第一个行为树

**步骤1：创建新文件**

- 点击工具栏的【新建】按钮
- 或按快捷键 `Ctrl + N`

**步骤2：添加节点**

- 从左侧节点面板点击需要的节点
- 节点会自动添加到画布中央

**步骤3：连接节点**

- 从父节点底部的输出端口拖拽到子节点顶部的输入端口
- 释放鼠标完成连接

**步骤4：配置属性**

- 点击选中节点
- 在右侧属性面板配置参数

**步骤5：保存运行**

- 点击【保存】按钮保存文件
- 点击【运行】按钮执行行为树

## 4. 节点类型详解

### 4.1 开始节点

**用途**：行为树的根节点，作为整个行为树的入口点，控制整体执行流程。

**特性**：

| 特性   | 说明                   |
| ---- | -------------------- |
| 自动创建 | 新建项目时自动创建，无需手动添加     |
| 不可删除 | 开始节点受保护，无法被删除、复制或剪切  |
| 失败继续 | 子节点失败后继续执行后续子节点（不短路） |
| 循环控制 | 支持重复执行次数控制           |

**执行流程**：

```
        开始
          │
   ┌─────┼─────┐
   ↓     ↓     ↓
子节点1 子节点2 子节点3 ...
   ↓     ↓     ↓
   └─────┼─────┘
         ↓
    重复/完成
```

**参数**：

| 参数     | 说明                            |
| ------ | ----------------------------- |
| 失败重试次数 | 子节点失败后的重试次数（-1 表示无限重试，默认 0）   |
| 重复次数   | 整个行为树的重复执行次数（-1 表示无限循环，默认 -1） |
| 重复间隔   | 重复执行之间的间隔时间（毫秒，默认 100）        |
| 超时时间   | 超时时间（毫秒，0 表示不限时）              |

**重复次数说明**：

| 次数    | 说明                      |
| ----- | ----------------------- |
| -1    | 无限循环（默认），行为树会一直执行直到手动停止 |
| 0     | 不重复，执行一次后结束             |
| N > 0 | 执行 N 次后结束               |

**使用场景**：

- 作为行为树的入口点
- 控制整体执行流程和重复次数
- 实现持续运行的自动化任务

**示例**：

```
开始节点（重复次数=-1）
├── OCR检测：检测目标
├── 鼠标点击：点击目标
└── 延时：等待1秒
```

**注意事项**：

- 每个行为树有且仅有一个开始节点
- 开始节点会自动创建在画布中央偏上的位置
- 无法通过节点面板手动添加开始节点

***

### 4.2 组合节点

组合节点是行为树的核心控制结构，包含多个子节点并按特定策略执行。

#### 4.2.1 顺序节点

**用途**：按顺序依次执行子节点，所有子节点成功才返回成功，任一失败立即返回失败。

**执行流程**：

```
          顺序
           │
   ┌───────┼────────┐
   ↓       ↓        ↓
子节点1 → 子节点2 → 子节点3 → ...
  ↓失败     ↓失败     ↓失败
返回失败   返回失败   返回失败
```

**参数**：

| 参数      | 说明                  |
| ------- | ------------------- |
| 子节点间隔   | 每个子节点执行完成后的等待时间（毫秒） |
| 失败后继续执行 | 开启后，中间节点失败仍继续执行后续节点 |

**使用场景**：

- 需要按固定顺序执行的任务
- 流程化的自动化操作
- 登录流程、表单填写等

**示例**：

```
顺序节点
├── OCR检测：检测登录按钮
├── 鼠标点击：点击登录按钮
├── 延时：等待2秒
└── OCR检测：检测登录成功
```

#### 4.2.2 选择节点

**用途**：按顺序执行子节点，任一成功即返回成功，全部失败才返回失败。

**执行流程**：

```
子节点1 → 子节点2 → 子节点3 → ...
  ↓成功     ↓成功     ↓成功
返回成功   返回成功   返回成功
```

**参数**：

| 参数    | 说明                  |
| ----- | ------------------- |
| 子节点间隔 | 每个子节点执行完成后的等待时间（毫秒） |

**使用场景**：

- 多种方案尝试，任一成功即可
- 容错处理，提供备选方案

**示例**：

```
选择节点
├── OCR检测：检测账号密码登录入口
├── OCR检测：检测扫码登录入口
└── OCR检测：检测第三方登录入口
```

#### 4.2.3 并行节点

**用途**：同时执行所有子节点，根据成功策略决定返回条件。

**执行流程**：

```
       并行
         │
   ┌─────┼─────┐
   ↓     ↓     ↓
子节点1 子节点2 子节点3
   ↓     ↓     ↓
   └─────┼─────┘
         ↓
     汇总结果
```

**参数**：

| 参数   | 说明                              |
| ---- | ------------------------------- |
| 成功策略 | 全部成功：所有子节点成功才成功；任一成功：任一子节点成功即成功 |

**使用场景**：

- 同时监控多个条件
- 并行执行独立任务

### 4.3 条件节点

条件节点检测环境状态，返回 SUCCESS 或 FAILURE，不产生副作用。

#### 4.3.1 OCR检测节点

**用途**：检测屏幕指定区域的文字内容。

**参数**：

| 参数    | 说明                             |
| ----- | ------------------------------ |
| 检测区域  | 屏幕上的检测范围                       |
| 关键词   | 要检测的文字（多个用逗号分隔）                |
| 语言    | OCR识别语言（eng/chi\_sim/chi\_tra） |
| 位置变量名 | 保存检测位置的黑板变量名                   |

**语言支持**：

| 语言代码     | 说明   |
| -------- | ---- |
| eng      | 英文   |
| chi\_sim | 简体中文 |
| chi\_tra | 繁体中文 |

**使用技巧**：

- 关键词可以是多个，用逗号分隔
- 检测成功后会自动保存位置到黑板，供点击节点使用
- 中文识别已针对中文文字特点进行优化

#### 4.3.2 图像匹配节点

**用途**：检测屏幕指定区域是否包含目标图像。

**参数**：

| 参数    | 说明                  |
| ----- | ------------------- |
| 检测区域  | 屏幕上的检测范围            |
| 模板路径  | 目标图像文件路径            |
| 匹配阈值  | 匹配度百分比（0\~100），默认80 |
| 位置变量名 | 保存检测位置的黑板变量名        |

**获取模板图像**：

1. 点击【截图】按钮
2. 工具窗口自动最小化
3. 在屏幕上拖拽选择要截取的区域
4. 截图自动保存到项目目录

**使用技巧**：

- 阈值越高，匹配越精确
- 建议阈值设置在 80\~90 之间
- 模板图像应尽量清晰、特征明显

#### 4.3.3 颜色检测节点

**用途**：检测屏幕指定区域是否包含目标颜色。

**参数**：

| 参数    | 说明                |
| ----- | ----------------- |
| 检测区域  | 屏幕上的检测范围          |
| 目标颜色  | 要检测的颜色（RGB值）      |
| 容差    | 颜色允许的偏差范围（0\~100） |
| 最小像素数 | 最少匹配像素数           |
| 位置变量名 | 保存检测位置的黑板变量名      |

**使用技巧**：

- 容差越大，匹配越宽松
- 建议容差设置在 10\~30 之间

#### 4.3.4 数字比较节点

**用途**：识别屏幕上的数字并进行比较判断。

**参数**：

| 参数    | 说明                     |
| ----- | ---------------------- |
| 检测区域  | 屏幕上的检测范围               |
| 语言    | OCR识别语言                |
| 比较运算符 | 比较运算符（>、>=、<、<=、==、!=） |
| 比较值   | 用于比较的阈值                |
| 位置变量名 | 保存检测位置的黑板变量名           |

**使用技巧**：

- 识别成功后数字值保存到黑板变量 `last_number_value`
- 可配合变量判断节点实现复杂逻辑

#### 4.3.5 变量判断节点

**用途**：判断黑板变量是否满足指定条件。

**参数**：

| 参数  | 说明        |
| --- | --------- |
| 变量名 | 要判断的黑板变量名 |
| 运算符 | 比较运算符     |
| 比较值 | 用于比较的值    |

**支持的运算符**：

| 运算符           | 说明          |
| ------------- | ----------- |
| ==            | 等于          |
| !=            | 不等于         |
| >             | 大于          |
| <             | 小于          |
| >=            | 大于等于        |
| <=            | 小于等于        |
| exists        | 变量存在        |
| not\_exists   | 变量不存在       |
| contains      | 包含（字符串/列表）  |
| not\_contains | 不包含（字符串/列表） |

### 4.4 动作节点

动作节点执行具体操作，产生副作用。

## 6. 连接操作

### 6.1 节点端口说明

每个节点有两个端口：

- **输入端口**：节点顶部的圆点，用于接收父节点的连接
- **输出端口**：节点底部的圆点，用于连接子节点

```
        ○ 输入端口
    ┌─────────┐
    │  节点    │
    └─────────┘
        ○ 输出端口
```

### 6.2 创建连接

1. 将鼠标移到父节点的**输出端口**（底部圆点）
2. 光标变为十字形状时，按住左键拖拽
3. 拖拽到子节点的**输入端口**（顶部圆点）
4. 释放鼠标完成连接

### 6.3 删除连接

| 方法  | 操作              |
| --- | --------------- |
| 快捷键 | 选中连线后按 `Delete` |

### 6.4 连接规则

- 一个输出端口可以连接多个子节点
- 一个输入端口只能连接一个父节点
- 不能创建循环连接（子节点连接回父节点）

### 6.5 执行顺序

当组合节点有多个子节点时，连接线上会显示序号（1、2、3...）表示执行顺序。

## 9. 黑板系统

### 9.1 什么是黑板？

黑板是节点间通信的核心机制，采用观察者模式，允许节点间解耦通信。

### 9.2 内置变量

| 变量名                       | 说明           |
| ------------------------- | ------------ |
| last\_detection\_position | 最近一次检测到的位置坐标 |
| last\_number\_value       | 最近一次识别的数字值   |
| execution\_count          | 执行计数         |

### 9.3 使用方式

**写入变量**：

- 条件节点检测成功后自动写入位置
- 设变量节点主动设置变量值

**读取变量**：

- 点击节点使用"点击最近检测点"功能
- 变量判断节点判断变量值

### 9.4 典型通信场景

**场景：OCR检测 → 点击检测位置**

```
OCRConditionNode                    MouseClickNode
     │                                    │
     │ 1. 检测关键词                       │
     │ 2. blackboard.set(                 │
     │    "last_detection_position", pos) │
     │ ──────────────────────────────────►│
     │                                    │ 3. blackboard.get()
     │                                    │ 4. 点击该位置
```

## 14. 常见问题

### Q1: 如何实现循环执行？

**A**: 设置开始或组合节点的"重复次数"为 -1 即可实现无限循环。

### Q2: 点击位置不正确怎么办？

**A**:

1. 检查检测区域是否正确
2. 确保勾选了"点击最近检测点"
3. 检测节点会保存绝对坐标，点击节点会使用该坐标

### Q3: OCR识别不准确怎么办？

**A**:

1. 确保检测区域只包含需要识别的文字
2. 尝试更换 OCR 语言设置
3. 中文识别已针对中文特点优化

### Q4: 图像匹配失败怎么办？

**A**:

1. 降低匹配阈值（如从 90 降到 80）
2. 确保模板图像清晰
3. 检查检测区域是否正确

### Q5: 如何使用"点击最近检测点"？

**A**:

1. 在条件节点（OCR/图像/颜色）中检测目标
2. 检测成功后位置会自动保存到黑板
3. 在点击节点中勾选"点击最近检测点"
4. 点击时会使用最近一次检测到的位置

### Q6: 如何实现长按操作？

**A**: 在点击节点中设置：

- 动作：press
- 按住时长：500（毫秒）

### Q7: 如何实现拖拽操作？

**A**: 使用鼠标移动节点的拖拽功能：

1. 设置移动类型=拖拽
2. 设置位置为拖拽起点
3. 设置拖拽终点为目标位置
4. 设置拖拽时长（建议 500-1000 毫秒）

### Q8: 如何实现无限点击？

**A**: 在点击节点中设置：

- 点击次数：-1
- 点击间隔：100（毫秒）
- 节点会持续点击，直到按 Escape 键停止

### Q9: 节点执行顺序如何确定？

**A**:

- 连接线上会显示序号（1、2、3...）
- 按序号顺序执行
- 如需调整，删除连接线后重新按期望顺序连接

### Q10: 如何分享项目？

**A**:

1. 点击【导出】按钮导出为 ZIP 文件
2. 将 ZIP 文件分享给其他用户
3. 接收者解压后打开项目文件夹即可使用
