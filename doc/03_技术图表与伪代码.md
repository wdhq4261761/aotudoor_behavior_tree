**日志级别说明：**

| 级别 | 颜色 | 用途 | 示例 |
|------|------|------|------|
| SUCCESS | 绿色 | 节点执行成功 | `[OCR检测节点] 检测登录按钮: 成功` |
| FAILURE | 红色 | 节点执行失败 | `[图像检测节点] 查找按钮: 失败 - 未找到匹配的图像，最高置信度: 45.23%` |

| INFO | 蓝色 | 一般信息 | `[系统] 行为树开始执行` |

**重要：** `_reset_for_retry()` 和 `_reset_for_repeat()` 必须将 `status` 设置为 `RUNNING`，而非 `SUCCESS`。如果设置为 `SUCCESS`，会导致：
- ConditionNode 在检测间隔内返回错误状态
- 装饰器无法正确判断节点是否需要继续执行

## 3 关键算法伪代码

### 3.0 节点日志输出算法

```
Algorithm: NodeLogging
Input: node, status, reason (optional)
Output: log entry added to LogManager

function log_node_result(node, status, reason=""):
    log_manager = LogManager.instance()
    
    switch status:
        case SUCCESS:
            log_manager.log_success(
                node_type=node.NODE_TYPE_NAME,
                node_name=node.name
            )
        
        case FAILURE:
            log_manager.log_failure(
                node_type=node.NODE_TYPE_NAME,
                node_name=node.name,
                reason=reason
            )
        
        case ABORTED:
            log_manager.log_aborted(
                node_type=node.NODE_TYPE_NAME,
                node_name=node.name
            )

// 失败原因生成规则
function generate_failure_reason(node_type, context):
    switch node_type:
        case "OCR检测节点":
            return f"未找到关键词: '{keywords}'，识别到的文本: {all_text}"
        
        case "图像检测节点":
            return f"未找到匹配的图像，最高置信度: {confidence:.2%}，阈值: {threshold:.2%}"
        
        case "数字节点":
            if all_text:
                return f"未识别到数字，识别到的文本: {all_text}"
            return "未识别到数字"
        
        case "变量条件节点":
            if not variable_exists:
                return f"变量 '{variable_name}' 不存在"
            return f"变量 '{variable_name}' 值为 {value}，不满足 {operator} {target_value}"
        
        case "并行节点":
            return f"成功 {success_count}/{enabled_count} 个子节点"
        
        case "开始节点":
            return "部分子节点执行失败"
        
        case "顺序节点":
            return f"子节点 [{failed_child_name}] 执行失败"
        
        case "选择节点":
            return "所有子节点执行失败"
        
        default:
            return "执行失败"
```

**成功日志简化原则：**

成功日志仅输出节点类型和名称，不包含额外信息：

```
// 正确示例
LogManager.instance().log_success(
    node_type="鼠标移动节点",
    node_name="移动到按钮位置"
)

// 错误示例（不应包含额外信息）
LogManager.instance().log_success(
    node_type="鼠标移动节点",
    node_name="移动到按钮位置",
    reason="位置: (100, 200)"  // 不应在成功日志中包含
)
```

### 3.1 行为树 Tick 算法

```
Algorithm: BehaviorTreeTick
Input: root_node, context
Output: NodeStatus

function tick_tree(root_node, context):
    context.tick_count += 1
    context.elapsed_time = current_time - start_time
    
    status = root_node.tick(context)
    
    if status != RUNNING:
        engine._running = false
        notify_status_change("completed", status)
    
    return status
```

### 3.2 Sequence 节点 Tick 算法

```
Algorithm: SequenceTick
Input: children[], current_index, context
Output: NodeStatus

function sequence_tick(children, current_index, context):
    if children is empty:
        return SUCCESS
    
    while current_index < len(children):
        child = children[current_index]
        
        // 跳过禁用节点
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        switch status:
            case RUNNING:
                return RUNNING    // 暂停，等待下次tick
            case FAILURE:
                current_index = 0 // 重置索引
                return FAILURE    // 短路返回
            case SUCCESS:
                current_index += 1  // 继续下一个
        
    // 所有子节点成功
    current_index = 0
    return SUCCESS
```

### 3.3 Parallel 节点 Tick 算法

```
Algorithm: ParallelTick
Input: children[], cached_statuses, success_policy, context
Output: NodeStatus

function parallel_tick(children, cached_statuses, success_policy, context):
    if children is empty:
        log_success(node_type="并行节点", node_name=self.name)
        return SUCCESS
    
    success_count = 0
    failure_count = 0
    running_children = []
    enabled_count = 0
    
    for i, child in enumerate(children):
        if not child.config.enabled:
            continue
        enabled_count += 1
        
        // 跳过已完成的子节点
        if i in cached_statuses:
            status = cached_statuses[i]
            if status == SUCCESS: success_count += 1; continue
            if status == FAILURE: failure_count += 1; continue
        
        // Tick 未完成的子节点
        status = child.tick(context)
        
        switch status:
            case SUCCESS:
                cached_statuses[i] = SUCCESS
                success_count += 1
            case FAILURE:
                cached_statuses[i] = FAILURE
                failure_count += 1
            case RUNNING:
                running_children.append(child)
    
    // require_one 策略：任一成功即返回
    if success_policy == REQUIRE_ONE and success_count > 0:
        abort_running(running_children)
        log_success(node_type="并行节点", node_name=self.name)
        return SUCCESS
    
    // 仍有运行中的子节点
    if running_children is not empty:
        return RUNNING
    
    // 所有子节点执行完毕
    if success_policy == REQUIRE_ALL:
        if success_count == enabled_count:
            log_success(node_type="并行节点", node_name=self.name)
            return SUCCESS
        else:
            log_failure(
                node_type="并行节点",
                node_name=self.name,
                reason=f"成功 {success_count}/{enabled_count} 个子节点"
            )
            return FAILURE
    else:
        if success_count > 0:
            log_success(node_type="并行节点", node_name=self.name)
            return SUCCESS
        else:
            log_failure(
                node_type="并行节点",
                node_name=self.name,
                reason=f"成功 {success_count}/{enabled_count} 个子节点"
            )
            return FAILURE
```

- 跳过 `enabled = False` 的子节点

function find_template(source, template, threshold):
    // 转换为 OpenCV 格式
    source_cv = cv2.cvtColor(np.array(source), cv2.COLOR_RGB2BGR)
    template_cv = cv2.cvtColor(np.array(template), cv2.COLOR_RGB2BGR)
    
    // 模板匹配
    result = cv2.matchTemplate(source_cv, template_cv, cv2.TM_CCOEFF_NORMED)
    
    // 获取最大匹配值（置信度）
    min_val, max_val, min_loc, max_loc = cv2.minMaxLoc(result)
    
    // 判断是否匹配成功
    if max_val >= threshold:
        // 计算中心点坐标
        h, w = template_cv.shape[:2]
        center_x = max_loc[0] + w // 2
        center_y = max_loc[1] + h // 2
        return (True, (center_x, center_y), max_val)
    
    // 匹配失败，返回最高置信度用于日志输出
    return (False, None, max_val)

// 使用示例
function image_condition_check(context):

            node_type="图像检测节点",
            node_name=self.name,
            reason=f"未找到匹配的图像，最高置信度: {confidence:.2%}，阈值: {threshold:.2%}"
        )

### 3.4 撤销/重做算法

```
Algorithm: CommandManager
State: undo_stack, redo_stack, max_history=100, is_executing=false

function execute(command):
    if is_executing: return false
    is_executing = true
    try:
        if command.execute():
            undo_stack.push(command)
            redo_stack.clear()
            if len(undo_stack) > max_history:
                undo_stack.pop_oldest()
            return true
        return false
    finally:
        is_executing = false

function undo():
    if undo_stack is empty: return false
    is_executing = true
    try:
        command = undo_stack.pop()
        if command.undo():
            redo_stack.push(command)
            return true
        undo_stack.push(command)  // 恢复失败，放回
        return false
    finally:
        is_executing = false

function redo():
    if redo_stack is empty: return false
    is_executing = true
    try:
        command = redo_stack.pop()
        if command.redo():
            undo_stack.push(command)
            return true
        redo_stack.push(command)  // 重做失败，放回
        return false
    finally:
        is_executing = false
```

        if node_id in _executor_pool:
            executor = _executor_pool[node_id]
            if executor.is_running:
                return executor  // 返回正在运行的执行器

        return executor

        // 停止当前执行器
        if _executor and _executor.is_running:
            _executor.stop_script()
        
        // 从执行器池中移除
        with _pool_lock:
            if node_id in _executor_pool:
                delete _executor_pool[node_id]
        
        _executor = None
        _script_started = False
        _aborted = False
    
    // 调用父类重置
    super.reset(reset_counters=reset_counters)

// 执行脚本

    executor = _get_or_create_executor()
    
    // 循环执行逻辑：loop=true 且 repeat_count=0
    use_loop = loop and config.repeat_count == 0
    
    if not _script_started:
        _script_started = True
        executor.run_script(_script_content, loop=use_loop)
    
    if executor.is_running:
        return RUNNING
    elif executor.last_error:
        return FAILURE
    else:
        return SUCCESS

```
Algorithm: NonBlockingDelay

function execute_delay_command(command):
    delay_time = command["time"] / 1000  // ms → s
    elapsed = 0
    
    // 分段检查 is_running，支持中断
    while elapsed < delay_time and is_running:
        sleep_time = min(0.1, delay_time - elapsed)  // 最多睡 100ms
        sleep(sleep_time)

    if not is_running:

    return true

### 3.10 反序列化算法

```
Algorithm: Deserialize
Input: data (Dict)

function deserialize(data):
    nodes_data = data["nodes"]
    connections = data["connections"]
    root_id = data["root_node"]

### 3.11 自动布局算法

```
Algorithm: AutoLayout
Input: nodes_data, root_id
Output: positions Dict[node_id → (x, y)]

const NODE_WIDTH = 180
const NODE_HEIGHT = 80
const H_GAP = 50
const V_GAP = 100

function auto_layout(nodes_data, root_id):
    positions = {}
    
    function get_children(node_id):
        return nodes_data[node_id]["children"]
    
    function calc_subtree_width(node_id):
        children = get_children(node_id)
        if children is empty:
            return NODE_WIDTH
        total = sum(calc_subtree_width(c) for c in children)
        return max(total, NODE_WIDTH)
    
    function layout_node(node_id, x, y):
        positions[node_id] = (x, y)
        children = get_children(node_id)
        if children is not empty:
            total_width = sum(calc_subtree_width(c) for c in children)
            current_x = x - total_width / 2
            for child in children:
                child_width = calc_subtree_width(child)
                layout_node(child, current_x + child_width/2, y + NODE_HEIGHT + V_GAP)
                current_x += child_width
    
    layout_node(root_id, 400, 100)
    return positions
```

### 3.12 项目导入算法

```
Algorithm: ImportProject
Input: zip_path (str), target_dir (str)
Output: Tuple[success: bool, error_msg: str, project_path: Optional[str]]

function import_project(zip_path, target_dir):
    // Step 1: 验证 ZIP 文件
    if not os.path.exists(zip_path):
        return (false, "文件不存在", null)
    
    if not zipfile.is_zipfile(zip_path):
        return (false, "不是有效的 ZIP 文件", null)
    
    // Step 2: 验证项目结构
    with zipfile.ZipFile(zip_path, 'r') as zipf:
        namelist = zipf.namelist()
        
        if namelist is empty:
            return (false, "ZIP 文件为空", null)
        
        found_project_json = false
        found_tree_json = false
        
        for name in namelist:
            basename = os.path.basename(name)
            if basename == "project.json":
                found_project_json = true
            elif basename == "tree.json":
                found_tree_json = true
        
        if not found_project_json:
            return (false, "缺少 project.json 文件", null)
        
        if not found_tree_json:
            return (false, "缺少 tree.json 文件", null)
    
    // Step 3: 获取项目名称
    project_name = get_project_name(zip_path)
    if not project_name:
        project_name = os.path.splitext(os.path.basename(zip_path))[0]
    
    project_root = os.path.join(target_dir, project_name)
    
    // Step 4: 检查同名冲突
    if os.path.exists(project_root):
        return (false, "PROJECT_EXISTS", project_root)
    
    // Step 5: 解压项目
    try:
        os.makedirs(target_dir, exist_ok=True)
        
        with zipfile.ZipFile(zip_path, 'r') as zipf:
            zip_root = get_project_root_in_zip(zip_path)
            
            for member in zipf.namelist():
                if zip_root:
                    if not member.startswith(zip_root + "/"):
                        continue
                    relative_path = member[len(zip_root) + 1:]
                else:
                    relative_path = member
                
                if not relative_path or relative_path.endswith("/"):
                    continue
                
                target_path = os.path.join(project_root, relative_path)
                os.makedirs(os.path.dirname(target_path), exist_ok=True)
                
                with zipf.open(member) as source:
                    with open(target_path, 'wb') as target:
                        target.write(source.read())
        
        return (true, "", project_root)
        
    except Exception as e:
        return (false, f"解压失败: {str(e)}", null)

// 获取 ZIP 内项目根目录
function get_project_root_in_zip(zip_path):
    with zipfile.ZipFile(zip_path, 'r') as zipf:
        for name in zipf.namelist():
            if name.endswith("project.json"):
                parts = name.rsplit("/", 1)
                if len(parts) > 1:
                    return parts[0]
                return ""
    return null

// 生成不冲突的新名称
function generate_new_name(base_name, target_dir):
    if not os.path.exists(os.path.join(target_dir, base_name)):
        return base_name
    
    counter = 1
    while true:
        new_name = f"{base_name}_{counter}"
        if not os.path.exists(os.path.join(target_dir, new_name)):
            return new_name
        counter += 1
        
        if counter > 1000:
            timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
            return f"{base_name}_{timestamp}"
```

### 3.12 Threshold 参数兼容性转换算法

```
Algorithm: ThresholdConversion
Input: threshold_value (float or int or str)
Output: normalized_threshold (float, range 0.0-1.0)

function normalize_threshold(threshold_value):
    // 转换为浮点数
    if isinstance(threshold_value, str):
        threshold = float(threshold_value)
    else:
        threshold = float(threshold_value)
    
    // 兼容性转换：百分比格式 -> 小数格式
    // 旧项目 (autodoor) 使用百分比格式，如 65 表示 65%
    // 新项目使用小数格式，如 0.65 表示 65%
    if threshold > 1.0:
        threshold = threshold / 100.0
    
    return threshold

// 示例:
// normalize_threshold(65)    → 0.65  (旧项目百分比格式)
// normalize_threshold("65")  → 0.65  (字符串百分比格式)

### 6.2 快捷键绑定表

**重要：快捷键必须绑定到顶级窗口**

使用 `root.bind()` 而非 `self.bind()`，确保在画布获得焦点时快捷键仍能生效：

```
Algorithm: BindShortcuts
Input: shortcuts (List of (key, callback) pairs)

function bind_shortcuts(shortcuts):
    root = get_toplevel()
    
    for key, callback in shortcuts:
        handler = make_handler(callback, key)
        root.bind(key, handler)

function make_handler(callback, key):
    def handler(event):
        if key in ["<Delete>", "<BackSpace>"]:
            focused = get_toplevel().focus_get()
            if focused:
                widget_type = get_class_name(focused)
                if widget_type in ["CTkEntry", "Entry", "CTkTextbox", "Text"]:
                    return None  // 允许输入控件正常处理
        
        if is_callable(callback):

### 11.2 拖拽阈值检测算法

```
Algorithm: OnNodeDrag
Input: event (MouseEvent)
Output: None

function on_drag(event):
    x, y = convert_to_world_coordinates(event)
    
    // 检查是否应该触发拖拽
    if not _dragging and _click_pos and _click_node_id:
        // 检查鼠标按钮是否仍然按下
        mouse_pressed = event.state & 0x0100
        if not mouse_pressed:
            // 鼠标已释放，清除状态
            _click_pos = null
            _click_node_id = null
            return
        
        // 计算移动距离
        dx = x - _click_pos[0]
        dy = y - _click_pos[1]
        distance = sqrt(dx*dx + dy*dy)
        
        // 移动超过阈值才触发拖拽
        if distance >= DRAG_THRESHOLD:
            if _click_node_id in nodes and _click_node_id in selected_nodes:
                _dragging = true
                _drag_node = _click_node_id
                node = nodes[_click_node_id]
                _drag_start = (_click_pos[0] - node.x, _click_pos[1] - node.y)
                _drag_start_pos = (node.x, node.y)
                _drag_start_positions = {}
                for each nid in selected_nodes:
                    if nid in nodes:
                        n = nodes[nid]
                        _drag_start_positions[nid] = (n.x, n.y)
    
    // 执行拖拽逻辑
    if _dragging and _drag_node:
        // 再次检查鼠标按钮状态
        mouse_pressed = event.state & 0x0100
        if not mouse_pressed:
            // 鼠标已释放，清除拖拽状态
            _dragging = false
            _drag_node = null
            _click_pos = null
            _click_node_id = null
            return
        
        // 计算偏移量
        dx = x - _drag_start[0] - _drag_start_pos[0]
        dy = y - _drag_start[1] - _drag_start_pos[1]
        
        // 移动所有选中的节点
        if len(selected_nodes) > 1 and _drag_start_positions:
            for each node_id in selected_nodes:
                if node_id in nodes and node_id in _drag_start_positions:
                    start_x, start_y = _drag_start_positions[node_id]
                    nodes[node_id].move_to(start_x + dx, start_y + dy)
        else:
            node = nodes[_drag_node]
            node.move_to(x - _drag_start[0], y - _drag_start[1])
        
        _redraw_connections()
```

### 11.3 鼠标释放处理算法

```
Algorithm: OnNodeRelease
Input: event (MouseEvent)
Output: None

function on_release(event):
    x, y = convert_to_world_coordinates(event)
    
    // 处理连接操作
    if _connecting:
        for each node_id, node in nodes:
            if node_id != _connect_start_node and node.is_on_input_port(x, y):
                _finish_connecting(node_id)
                return
        _cancel_connecting()
        return
    
    // 处理框选操作
    if _selecting:
        _finish_selection(x, y)
        return
    
    // 处理拖拽完成
    if _dragging and _drag_node:
        // 触发节点移动回调
        if on_nodes_move:
            old_positions = {}
            new_positions = {}
            for each node_id in selected_nodes:
                if node_id in nodes and node_id in _drag_start_positions:
                    node = nodes[node_id]
                    old_positions[node_id] = _drag_start_positions[node_id]
                    new_positions[node_id] = (node.x, node.y)
            on_nodes_move(old_positions, new_positions)
    
    // 如果没有拖拽，则在释放时加载属性面板
    if not _dragging and _click_node_id:
        if _click_node_id in nodes and on_node_select:
            node = nodes[_click_node_id]
            on_node_select(_click_node_id, node.node_type)
    
    // 清除所有状态
    _dragging = false
    _drag_node = null
    _click_pos = null
    _click_node_id = null
    _panning = false
```

function import_resource(source_path, project_root):
    // 1. 检测资源类型
    resource_type = detect_resource_type(source_path)
    
    // 2. 确定目标目录
    target_dir = get_target_directory(resource_type)
    // resource_type → target_dir 映射:
    // "image" → "images/templates"
    // "script" → "scripts/script"
    // "code" → "scripts/code"
    // "audio" → "audio/alarms"
    // "data" → "data/config"
    
    // 3. 处理文件名冲突
    filename = basename(source_path)
    target_path = join(project_root, target_dir, filename)
    
    if exists(target_path):
        counter = 2
        while exists(target_path):
            name, ext = splitext(filename)
            new_filename = f"{name}_{counter}{ext}"
            target_path = join(project_root, target_dir, new_filename)
            counter += 1
        filename = new_filename
    
    // 4. 复制文件到项目目录
    create_directory(dirname(target_path))
    copy_file(source_path, target_path)
    
    // 5. 生成相对路径引用
    relative_path = "./" + target_dir + "/" + filename
    relative_path = relative_path.replace("\\", "/")
    
    return relative_path
```

### 12.5 路径解析流程

```
Algorithm: ResolvePath
Input: relative_path (str), project_root (str)
Output: absolute_path (str)

function resolve_path(relative_path, project_root):
    // 1. 检查是否为相对路径
    if not relative_path starts with "./":
        return relative_path  // 已经是绝对路径
    
    // 2. 移除 "./" 前缀
    path_without_prefix = relative_path[2:]
    
    // 3. 构建绝对路径
    absolute_path = join(project_root, path_without_prefix)
    
    // 4. 规范化路径
    absolute_path = normalize_path(absolute_path)
    
    // 5. 验证路径安全性
    real_path = realpath(absolute_path)
    real_project_root = realpath(project_root)
    
    if not real_path starts with real_project_root:
        raise Error("路径超出项目范围")
    
    return absolute_path
```

function create_backup(project_root):
    tree_file = join(project_root, "tree.json")
    
    if not exists(tree_file):
        return None
    
    // 1. 生成备份文件名
    timestamp = format_datetime(now(), "%Y%m%d_%H%M%S")
    backup_file = join(project_root, f"tree_backup_{timestamp}.json")

    copy_file(tree_file, backup_file)

    for file in list_files(project_root):
        if file starts with "tree_backup_" and file ends with ".json":
            backup_files.append(join(project_root, file))
    
    // 按修改时间排序（最新的在前）
    backup_files.sort_by(modification_time, descending=True)
    
    // 保留最近 5 个，删除其余的
    for old_backup in backup_files[5:]:
        delete_file(old_backup)
    
    return backup_file

### 12.7 ResourceService 资源管理服务

Algorithm: ResourceService

    'template_path': 'image',

const CACHE_DIR = 'cache'

function collect_external_resources(tree_data):
    external_resources = []
    nodes = tree_data.get('nodes', {})
    
    for node_id, node_data in nodes:
        for path_key, resource_type in RESOURCE_TYPE_MAP:
            path = node_data.get('config', {}).get(path_key)
            if path and not path.startswith('./'):
                external_resources.append({
                    'node_id': node_id,
                    'path_key': path_key,
                    'path': path,
                    'resource_type': resource_type
                })
    
    return external_resources

function import_resources_to_project(resources):
    path_mapping = {}
    
    for resource in resources:
        new_path = import_single_file_to_project(
            resource['path'],
            resource['resource_type']
        )
        if new_path:
            key = resource['node_id'] + '.' + resource['path_key']
            path_mapping[key] = new_path
    
    return path_mapping

function import_single_file_to_project(source_path, resource_type, old_path=null):
    target_dir = TYPE_DIR_MAP.get(resource_type, 'data/other')
    filename = basename(source_path)
    target_path = join(project_root, target_dir, filename)
    
    // 处理文件名冲突
    if exists(target_path):
        name, ext = splitext(filename)
        counter = 2
        while exists(target_path):
            filename = f"{name}_{counter}{ext}"
            target_path = join(project_root, target_dir, filename)
            counter += 1
    
    // 复制文件
    create_directory(dirname(target_path))
    copy_file(source_path, target_path)
    
    // 如果有旧文件，移动到缓存目录
    if old_path:
        move_to_cache(old_path)
    
    return resolver.to_relative(target_path)

function move_to_cache(file_path):
    if not file_path or file_path.startswith('./'):
        return false
    
    if not exists(file_path):
        return false
    
    cache_dir = join(project_root, CACHE_DIR)
    create_directory(cache_dir)
    
    filename = basename(file_path)
    name, ext = splitext(filename)
    timestamp = format_datetime(now(), "%Y%m%d_%H%M%S")
    cache_filename = f"{name}_{timestamp}{ext}"
    cache_path = join(cache_dir, cache_filename)
    
    move_file(file_path, cache_path)
    return true

function update_tree_paths(tree_data, path_mapping):
    nodes = tree_data.get('nodes', {})
    
    for key, new_path in path_mapping:
        node_id, path_key = rsplit(key, '.', 1)
        if node_id in nodes:
            if 'config' not in nodes[node_id]:
                nodes[node_id]['config'] = {}
            nodes[node_id]['config'][path_key] = new_path
    
    return tree_data
```
