# AutoDoor 行为树系统 — 开始节点设计方案

## 1 设计概述

### 1.1 功能定位

「开始」节点是行为树的根节点,负责控制整个行为树的执行流程。每个项目有且仅有一个开始节点,作为行为树的入口点。

### 1.2 核心特性

| 特性     | 说明                |
| ------ | ----------------- |
| 自动创建   | 新建项目时自动在画布上创建     |
| 不可删除   | 具备保护属性,禁止删除、复制、剪切 |
| 根节点角色  | 作为行为树的唯一入口点       |
| 重复执行控制 | 支持无限循环或固定次数执行     |
| 失败继续执行 | 子节点失败后继续执行后续子节点   |

### 1.3 设计原则

- **复用现有架构** - 继承SequenceNode,避免重复造轮子
- **最小改动原则** - 在现有节点体系上扩展,减少对现有代码的影响
- **架构一致性** - 与现有节点类型保持一致的设计模式

## 2 节点数据结构定义

### 2.1 类定义

```python
# bt_core/nodes.py

class StartNode(SequenceNode):
    """
    开始节点 - 行为树的根节点
    
    特性:
    - 继承SequenceNode的顺序执行逻辑
    - 失败后继续执行(不短路)
    - 支持重复执行次数控制
    - 不可删除、不可复制、不可剪切
    """
    NODE_TYPE = "StartNode"
    
    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self.repeat_count = self.config.get_int("repeat_count", -1)  # -1表示无限循环
        self._current_repeat = 0  # 当前重复次数
        self._is_protected = True  # 不可删除标记
```

### 2.2 核心属性说明

| 属性                | 类型   | 默认值         | 说明               |
| ----------------- | ---- | ----------- | ---------------- |
| NODE\_TYPE        | str  | "StartNode" | 节点类型标识           |
| repeat\_count     | int  | -1          | 重复执行次数(-1表示无限循环) |
| \_current\_repeat | int  | 0           | 当前已执行次数          |
| \_is\_protected   | bool | True        | 节点保护标记           |

### 2.3 继承关系

```
Node (ABC)
└── CompositeNode (ABC)
    └── SequenceNode
        └── StartNode
            ├── 继承顺序执行逻辑
            ├── 重写失败处理逻辑(失败后继续)
            └── 添加重复执行控制
```

## 3 核心执行逻辑

### 3.1 Tick方法实现

```python
def tick(self, context: ExecutionContext) -> NodeStatus:
    """
    执行所有子节点,失败后继续执行
    
    与SequenceNode的区别:
    - SequenceNode: 任一子节点失败立即返回FAILURE
    - StartNode: 子节点失败后继续执行后续子节点
    
    Args:
        context: 执行上下文
        
    Returns:
        NodeStatus: 执行状态
    """
    if not self.children:
        return NodeStatus.SUCCESS
    
    # 执行所有子节点(不短路)
    has_running = False
    for child in self.children:
        if not child.config.enabled:
            continue
        
        status = child.tick(context)
        
        # 如果有子节点正在运行,记录状态但不中断执行
        if status == NodeStatus.RUNNING:
            has_running = True
    
    # 如果有子节点正在运行,返回RUNNING
    if has_running:
        return NodeStatus.RUNNING
    
    # 所有子节点执行完毕,处理重复逻辑
    self._current_repeat += 1
    
    if self.repeat_count == -1:
        # 无限循环模式
        self.reset()
        return NodeStatus.RUNNING
    elif self._current_repeat < self.repeat_count:
        # 继续重复
        self.reset()
        return NodeStatus.RUNNING
    else:
        # 达到重复次数,执行完成
        return NodeStatus.SUCCESS
```

### 3.2 Reset方法实现

```python
def reset(self) -> None:
    """重置节点状态"""
    super().reset()
    # 重置所有子节点
    for child in self.children:
        child.reset()
```

### 3.3 执行流程图

```
开始执行
    │
    ├─ 检查子节点列表
    │   └─ 无子节点 → 返回SUCCESS
    │
    ├─ 遍历执行所有子节点
    │   ├─ 跳过禁用的子节点
    │   ├─ 执行子节点.tick()
    │   └─ 记录RUNNING状态(不中断)
    │
    ├─ 检查是否有RUNNING状态
    │   └─ 有 → 返回RUNNING
    │
    ├─ 所有子节点执行完毕
    │   └─ _current_repeat += 1
    │
    ├─ 检查重复次数
    │   ├─ repeat_count == -1 → reset() → 返回RUNNING
    │   ├─ _current_repeat < repeat_count → reset() → 返回RUNNING
    │   └─ 达到重复次数 → 返回SUCCESS
    │
    └─ 结束
```

## 4 自动创建逻辑

### 4.1 创建时机

在以下场景自动创建开始节点:

1. **新建项目** - 用户点击"新建"按钮或使用快捷键Ctrl+N
2. **清空画布** - 用户点击"清空"按钮后重新初始化

### 4.2 创建流程

```python
# bt_gui/bt_editor/editor.py

def new_tree(self):
    """新建项目时自动创建开始节点"""
    # 清空画布
    self.canvas.clear_canvas()
    
    # 创建开始节点
    start_node = StartNode()
    start_node.config.name = "开始"
    
    # 计算默认位置
    # 获取画布尺寸
    canvas_width = self.canvas.winfo_width()
    canvas_height = self.canvas.winfo_height()
    
    # 如果画布尺寸为0(尚未渲染),使用默认值
    if canvas_width <= 0:
        canvas_width = 800
    if canvas_height <= 0:
        canvas_height = 600
    
    # 左右居中
    x = canvas_width / 2
    
    # 上下位置偏顶部,预留顶部空间(约20%的高度)
    y = canvas_height * 0.05
    
    # 构建节点数据
    node_data = {
        "id": start_node.node_id,
        "type": "StartNode",
        "name": "开始",
        "enabled": True,
        "config": {
            "repeat_count": -1
        },
        "position": {
            "x": x,
            "y": y
        }
    }
    
    # 添加到画布
    self.canvas.add_node(node_data)
    
    # 设置为根节点
    self.canvas.root_node_id = start_node.node_id
```

### 4.3 位置计算说明

```
画布坐标系:
┌────────────────────────────────────────┐
│                                        │
│         预留空间 (5%)                  │
│                                        │
│         ┌─────────────┐                │
│         │   开始节点   │  ← y = height * 0.05
│         └─────────────┘
│                │                       │
│                │ x = width / 2         │
│                ▼                       │
│         ┌─────────────┐                │
│         │  子节点区域  │                │
│         └─────────────┘                │
│                                        │
│                                        │
└────────────────────────────────────────┘
```

**位置计算公式:**

- x坐标 = 画布宽度 / 2 (左右居中)
- y坐标 = 画布高度 \* 0.05 (顶部预留5%空间)

## 5 不可删除机制实现

### 5.1 方案选择

采用**方案A: 在节点层面标记**,在节点基类中添加保护标记,在画布和编辑器层面检查该标记。

### 5.2 节点基类扩展

```python
# bt_core/nodes.py

class Node(ABC):
    """节点抽象基类"""
    
    NODE_TYPE = "Node"
    
    def __init__(self, node_id=None, config=None):
        self.node_id = node_id or str(uuid4())[:8]
        self.config = config or NodeConfig()
        self.status = NodeStatus.SUCCESS
        self.children: List[Node] = []
        self.parent: Optional[Node] = None
        self._tick_count = 0
        self._is_protected = False  # 新增: 节点保护标记
    
    def is_protected(self) -> bool:
        """
        检查节点是否受保护(不可删除)
        
        Returns:
            bool: True表示受保护,False表示可删除
        """
        return self._is_protected
```

### 5.3 画布删除检查

```python
# bt_gui/bt_editor/canvas.py

def _delete_selected(self):
    """删除选中的节点"""
    nodes_to_delete = []
    protected_nodes = []
    
    for node_id in self.selected_nodes:
        node_item = self.nodes.get(node_id)
        if node_item:
            # 获取节点实例
            node = node_item.node
            if node and node.is_protected():
                # 记录受保护的节点
                protected_nodes.append(node_id)
                continue
            nodes_to_delete.append(node_id)
    
    # 如果有受保护的节点,显示提示
    if protected_nodes:
        from tkinter import messagebox
        messagebox.showwarning(
            "无法删除",
            f"开始节点不可删除"
        )
    
    # 删除非保护节点
    if nodes_to_delete:
        command = RemoveNodesCommand(self, nodes_to_delete)
        self.command_manager.execute(command)
```

### 5.4 禁止复制和剪切

```python
# bt_gui/bt_editor/editor.py

def _copy_selected(self):
    """复制选中的节点"""
    nodes_to_copy = []
    
    for node_id in self.canvas.selected_nodes:
        node_item = self.canvas.nodes.get(node_id)
        if node_item:
            node = node_item.node
            if node and node.is_protected():
                # 跳过受保护的节点
                continue
            nodes_to_copy.append(node_id)
    
    if nodes_to_copy:
        # 复制节点数据到剪贴板
        self._clipboard_data = self.canvas.get_nodes_data(nodes_to_copy)
    else:
        self._clipboard_data = None


def _cut_selected(self):
    """剪切选中的节点"""
    nodes_to_cut = []
    
    for node_id in self.canvas.selected_nodes:
        node_item = self.canvas.nodes.get(node_id)
        if node_item:
            node = node_item.node
            if node and node.is_protected():
                # 跳过受保护的节点
                continue
            nodes_to_cut.append(node_id)
    
    if nodes_to_cut:
        # 复制节点数据到剪贴板
        self._clipboard_data = self.canvas.get_nodes_data(nodes_to_cut)
        # 删除节点
        command = RemoveNodesCommand(self.canvas, nodes_to_cut)
        self.command_manager.execute(command)
    else:
        self._clipboard_data = None
```

class NodeItem:
    def _get_category_color(self) -> str:
        """获取节点分类颜色"""
        if self.node_type == "StartNode":
            return Theme.NODE_COLORS["start"]
        # ... 其他节点类型的颜色判断
    
    def _draw_node(self):
        """绘制节点"""
        # 绘制阴影
        self.canvas.create_rectangle(
            x + 3, y + 3, x + width + 3, y + height + 3,
            fill="#000000", outline="", tags=("node_shadow", self.node_id)
        )
        
        # 绘制主体
        color = self._get_category_color()
        self.canvas.create_rectangle(
            x, y, x + width, y + height,
            fill=self._dark_colors['bg_secondary'],
            outline=color,
            width=2,
            tags=("node", self.node_id)
        )
        
        # 绘制左侧颜色条
        self.canvas.create_rectangle(
            x, y, x + 4, y + height,
            fill=color, outline="",
            tags=("node_color", self.node_id)
        )
        
        # 绘制节点名称
        display_name = self.node_name
        if self.node_type == "StartNode":
            # 开始节点添加特殊图标
            display_name = "▶ " + display_name
        
        self.canvas.create_text(
            x + 15, y + height / 2,
            text=display_name,
            fill=self._dark_colors['text_primary'],
            font=(Theme.FONTS['family'], 10),
            anchor="w",
            tags=("node_text", self.node_id)
        )

## 7 序列化支持

### 7.1 序列化方法

```python
# bt_core/nodes.py

class StartNode(SequenceNode):
    def to_dict(self) -> Dict[str, Any]:
        """
        序列化为字典
        
        Returns:
            Dict[str, Any]: 节点数据字典
        """
        data = super().to_dict()
        data["repeat_count"] = self.repeat_count
        data["_is_protected"] = self._is_protected
        return data
```

### 7.2 反序列化方法

```python
@classmethod
def from_dict(cls, data: Dict[str, Any]) -> "StartNode":
    """
    从字典反序列化
    
    Args:
        data: 节点数据字典
        
    Returns:
        StartNode: 节点实例
    """
    node = super().from_dict(data)
    node.repeat_count = data.get("repeat_count", -1)
    node._is_protected = data.get("_is_protected", True)
    return node
```

def _delete_selected(self):
    nodes_to_delete = []
    protected_nodes = []
    
    for node_id in self.selected_nodes:
        node_item = self.nodes.get(node_id)
        if node_item and node_item.node:
            if node_item.node.is_protected():
                protected_nodes.append(node_id)
                continue
            nodes_to_delete.append(node_id)
    
    if protected_nodes:
        messagebox.showwarning("无法删除", "开始节点不可删除")
    
    if nodes_to_delete:
        command = RemoveNodesCommand(self, nodes_to_delete)
        self.command_manager.execute(command)

## 12 技术选型说明

### 12.1 为什么继承SequenceNode?

**理由:**

1. **代码复用** - SequenceNode已实现子节点管理、状态维护等核心逻辑
2. **架构一致** - 与现有复合节点保持一致的设计模式
3. **易于维护** - 减少重复代码,降低维护成本
4. **最小改动** - 只需重写失败处理逻辑和添加重复执行功能

**对比其他方案:**

| 方案              | 优点        | 缺点         |
| --------------- | --------- | ---------- |
| 继承SequenceNode  | 代码复用,架构一致 | 需要重写部分逻辑   |
| 继承CompositeNode | 更灵活       | 需要重新实现更多逻辑 |
| 继承Node基类        | 完全自主控制    | 代码重复最多     |
| 装饰器模式           | 灵活性最高     | 实现复杂度高     |

### 12.2 为什么使用节点层面标记?

**理由:**

1. **单一职责** - 节点自己知道是否可删除,符合面向对象原则
2. **易于扩展** - 未来可能有其他类型的保护节点
3. **集中管理** - 保护逻辑在节点类中统一管理
4. **序列化支持** - 保护状态可以持久化

**对比其他方案:**

| 方案     | 优点        | 缺点        |
| ------ | --------- | --------- |
| 节点层面标记 | 单一职责,易于扩展 | 需要修改基类    |
| 画布层面检查 | 不修改节点类    | 逻辑分散,难以维护 |
| 配置文件管理 | 集中配置      | 运行时检查复杂   |

### 12.3 为什么不在节点面板显示?

**理由:**

1. **唯一性** - 每个项目只能有一个开始节点
2. **自动创建** - 新建项目时已自动创建
3. **避免混淆** - 防止用户误添加多个开始节点
4. **简化UI** - 减少不必要的UI元素
