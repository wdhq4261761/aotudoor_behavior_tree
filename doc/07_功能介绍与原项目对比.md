# AutoDoor 行为树工具 — 功能介绍与原项目对比

## 文档说明

本文档介绍 AutoDoor 行为树独立项目的具体功能，并与原项目 `autodoor` 进行详细对比，阐述行为树项目的优势与适用场景。

---

## 1. 项目定位对比

### 1.1 原项目 (autodoor)

**定位**：多功能自动化工具箱

原项目是一个集成了多种自动化功能的综合性工具，采用**多标签页**布局，每个标签页对应一个独立的功能模块。

```
┌─────────────────────────────────────────────────────────────────┐
│                    原项目功能架构                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐              │
│  │ 首页    │ │文字识别 │ │定时功能 │ │数字识别 │              │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘              │
│                                                                 │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐              │
│  │图像检测 │ │后台监控 │ │脚本运行 │ │基本设置 │              │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘              │
│                                                                 │
│  各功能模块独立运行，通过首页统一控制启停                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 行为树项目 (autodoor_behavior_tree)

**定位**：可视化流程编排工具

行为树项目专注于**流程编排**，通过图形化界面拖拽节点、连接节点来创建复杂的自动化流程。

```
┌─────────────────────────────────────────────────────────────────┐
│                    行为树项目架构                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    行为树编辑器                           │   │
│  │                                                         │   │
│  │   ┌───────┐     ┌───────┐     ┌───────┐                │   │
│  │   │ 开始  │ ──► │ OCR   │ ──► │ 点击  │                │   │
│  │   │       │     │ 检测  │     │       │                │   │
│  │   └───────┘     └───────┘     └───────┘                │   │
│  │       │                                           │     │   │
│  │       └──────────────► ┌───────┐ ◄───────────────┘     │   │
│  │                        │ 延时  │                        │   │
│  │                        └───────┘                        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  通过节点连接构建完整流程，支持复杂逻辑控制                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 详细功能对比

### 3.1 文字识别功能

#### 原项目实现

```
文字识别标签页
├── 识别组管理（最多15个）
│   ├── 区域选择
│   ├── 识别间隔（1-30秒）
│   ├── 暂停时长（30-300秒）
│   ├── 触发按键
│   ├── 关键词设置
│   ├── OCR语言选择
│   └── 启用报警
└── 特点：独立运行，检测到关键词后执行固定操作
```

#### 行为树项目实现

```
OCR检测节点
├── 检测区域
├── 关键词
├── OCR语言
├── 位置变量名（黑板）
└── 装饰参数
    ├── 结果取反
    ├── 失败重试
    ├── 超时时间
    └── 检测间隔

配合其他节点使用：
OCR检测 → 鼠标点击（使用检测位置）→ 延时 → 其他操作...
```

#### 对比分析

| 方面 | 原项目 | 行为树项目 |
|-----|--------|-----------|
| 配置方式 | 表单配置 | 节点属性配置 |
| 触发后操作 | 固定按键/点击 | 可连接任意后续节点 |
| 流程控制 | 无 | 支持条件分支、循环 |
| 数据传递 | 无 | 通过黑板传递检测位置 |
| 灵活性 | 中等 | 高 |

### 3.2 定时功能

#### 原项目实现

```
定时功能标签页
├── 定时组管理（最多15个）
│   ├── 间隔时间
│   ├── 触发按键
│   ├── 按键时长
│   ├── 启用鼠标点击
│   ├── 点击位置
│   └── 启用报警
└── 特点：按固定间隔执行固定操作
```

#### 行为树项目实现

```
延时节点 + 装饰参数
├── 延时时长
└── 配合开始节点的重复参数
    ├── 重复次数（-1=无限）
    └── 重复间隔

示例流程：
开始（重复次数=-1，间隔=5000ms）
└── 顺序节点
    ├── 按键节点
    └── 鼠标点击节点
```

#### 对比分析

| 方面 | 原项目 | 行为树项目 |
|-----|--------|-----------|
| 定时方式 | 固定间隔循环 | 通过开始节点重复参数 |
| 执行内容 | 固定按键/点击 | 任意节点组合 |
| 多任务 | 支持多组并行 | 需要多个行为树 |
| 灵活性 | 中等 | 高 |

#### 原项目实现

├── 监控组管理（最多15个）
│   ├── 监控类型（OCR/图像/颜色）
│   ├── 监控区域

### 3.5 流程控制能力

#### 原项目

```
流程控制能力：无

各功能模块独立运行，无法实现：
• 条件判断后的不同分支
• 循环执行特定步骤
• 并行执行多个任务
• 复杂的逻辑组合
```

#### 行为树项目

```
流程控制能力：强大

组合节点提供完整流程控制：

1. 顺序节点（Sequence）
   按顺序执行，全部成功才成功
   
2. 选择节点（Selector）
   按顺序尝试，任一成功即成功
   
3. 并行节点（Parallel）
   同时执行所有子节点

示例：登录流程
选择节点（尝试多种登录方式）
├── 顺序节点（账号密码登录）
│   ├── OCR检测：检测账号输入框
│   ├── 点击：点击账号输入框
│   ├── 按键：输入账号
│   ├── OCR检测：检测密码输入框
│   ├── 点击：点击密码输入框
│   ├── 按键：输入密码
│   └── 点击：点击登录按钮
├── 顺序节点（扫码登录）
│   ├── OCR检测：检测二维码
│   └── 延时：等待扫码
└── 顺序节点（第三方登录）
    └── ...
```

### 3.6 数据共享能力

#### 原项目

```
数据共享：无

各功能模块之间无法共享数据：
• OCR检测的位置无法传递给其他模块
• 数字识别的结果无法用于条件判断
• 无法实现"检测A后执行B"的联动
```

#### 行为树项目

```
数据共享：黑板系统

黑板系统实现节点间数据共享：

内置变量：
• last_detection_position：最近检测位置
• last_number_value：最近识别数字
• execution_count：执行计数

使用示例：
OCR检测节点
    │
    │ 检测成功，位置保存到黑板
    │ blackboard.set("last_detection_position", pos)
    ▼
鼠标点击节点
    │
    │ 从黑板读取位置
    │ position = blackboard.get("last_detection_position")
    ▼
点击检测到的位置

自定义变量：
设变量节点 → 设置自定义变量
变量判断节点 → 判断变量值
```
