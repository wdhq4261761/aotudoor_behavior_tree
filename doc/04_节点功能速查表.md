# AutoDoor 行为树系统 — 节点功能总表

本文档提供所有节点的快速参考,每个节点一行,重点介绍特色功能。

***

## 复合节点（4个）

| 节点名称     | 功能说明                         | 特色参数                                              | 典型用途                |
| -------- | ---------------------------- | ------------------------------------------------- | ------------------- |
| **开始节点** | 作为根节点，顺序执行所有子节点，失败后继续执行后续子节点 | 无                                                 | 行为树的根节点，确保所有子节点都被执行 |
| **顺序节点** | 按顺序执行子节点，所有成功才成功，任一失败立即停止    | **失败后继续运行**：开启后会先把子节点都运行后，再根据运行情况返回成功/失败          | 需要严格按顺序执行的任务链       |
| **选择节点** | 按顺序执行子节点，任一成功即成功，全部失败才失败     | 无                                                 | 多种方案的尝试，失败后尝试备选方案   |
| **并行节点** | 同时执行所有子节点，根据策略决定成功条件         | **成功策略**: require\_all(全部成功) / require\_one(任一成功) | 需要同时执行的独立任务         |

***

## 条件节点（5个）

| 节点名称        | 功能说明                  | 特色参数                                                                     | 典型用途            |
| ----------- | --------------------- | ------------------------------------------------------------------------ | --------------- |
| **OCR检测节点** | 通过OCR识别屏幕文字，检测是否包含关键词 | **关键词**: 要检测的文字**语言**: eng/chi\_sim**位置变量**: 保存位置的黑板变量名                  | 检测界面文字，如按钮、提示信息 |
| **图像匹配节点**  | 通过模板匹配检测屏幕是否包含目标图像    | **模板路径**: 图像文件路径**匹配阈值**: 0.0-1.0**位置变量**: 保存位置的黑板变量名                    | 检测图标、按钮等图像元素    |
| **颜色检测节点**  | 检测屏幕是否包含目标颜色          | **目标颜色**: #RRGGBB格式**颜色容差**: 0-255**最小像素数**: 匹配像素数阈值**位置变量**: 保存位置的黑板变量名 | 检测血条、状态指示灯等颜色元素 |
| **数字比较节点**  | 通过OCR识别数字并与目标值比较      | **比较运算符**: >/>=/\</<=/==/!=**目标值**: 比较的数值**位置变量**: 保存检测位置的黑板变量名          | 检测血量、金币、等级等数值   |
| **变量判断节点**  | 判断黑板变量的值是否满足条件        | **变量名**: 要判断的变量**运算符**: ==/!=/>/>=/\</<=/contains/exists等**目标值**: 比较的值   | 根据变量值控制流程，循环计数  |

## 动作节点（9个）

| 节点名称       | 功能说明      | 特色参数                                                                                       | 典型用途             |
| ---------- | --------- | ------------------------------------------------------------------------------------------ | ---------------- |
| **按键节点**   | 执行键盘按键操作  | **按键名称**: enter/space/a/b等**按键动作**: press(按下抬起)/down(按下)/up(抬起)**按住时长**: 毫秒数               | 模拟键盘输入、快捷键       |
| **鼠标点击节点** | 执行鼠标点击操作  | **鼠标按键**: left/right/middle**点击位置**: (x,y)坐标，留空则使用上次检测位置**点击次数**: 1(单击)/2(双击)**按住时长**: 毫秒数 | 点击按钮、选择菜单        |
| **鼠标移动节点** | 移动鼠标到指定位置 | **目标位置**: (x,y)坐标**相对移动**: true(相对当前位置)/false(绝对坐标)**平滑移动**: 是否使用缓动函数                      | 移动鼠标到目标位置        |
| **鼠标滚动节点** | 执行鼠标滚轮滚动  | **滚动距离**: 像素数**滚动方向**: up/down                                                             | 滚动页面、列表          |
| **延时节点**   | 等待指定时间    | **延时时长**: 毫秒数                                                                              | 等待界面加载、动画播放      |
| **设置变量节点** | 操作黑板变量    | **变量名**: 要操作的变量**操作类型**: set(设置)/increment(递增)/delete(删除)**变量值**: 设置的值**递增量**: 每次递增的数量     | 设置计数器、标志位        |
| **脚本节点**   | 执行录制脚本文件  | **脚本路径**: 脚本文件路径**循环执行**: 是否循环(仅repeat\_count=0时有效)                                        | 执行预先录制的操作序列      |
| **代码节点**   | 执行外部代码文件  | **代码路径**: 代码文件路径**命令行参数**: 传递给代码的参数**等待完成**: 是否等待执行完成                                      | 执行Python脚本、批处理文件 |
| **报警节点**   | 播放报警音效    | **音效路径**: 音频文件路径**音量**: 0-100**等待完成**: 是否等待播放完成                                            | 提醒用户注意、异常报警      |

## 通用参数说明

所有节点都支持以下通用参数（不在特色参数中列出）：

| 参数名 | 说明     | 默认值        |
| --- | ------ | ---------- |
| 名称  | 节点名称   | 根据节点类型自动命名 |
| 启用  | 是否启用节点 | true       |
| 描述  | 节点描述   | 空          |

### 条件节点通用参数

| 参数名    | 说明         | 默认值   |
| ------ | ---------- | ----- |
| 结果取反   | 是否对检测结果取反  | false |
| 失败重试次数 | 检测失败时的重试次数 | 0     |
| 超时时间   | 超时时间（毫秒）   | 0     |
| 检测间隔   | 检测间隔（毫秒）   | 300   |

### 动作节点通用参数

| 参数名  | 说明          | 默认值 |
| ---- | ----------- | --- |
| 重复次数 | 重复执行的次数     | 0   |
| 重复间隔 | 重复执行的间隔（毫秒） | 100 |
| 超时时间 | 超时时间（毫秒）    | 0   |

***

## 黑板系统

节点间通过黑板系统进行通信：

| 内置变量                      | 说明           |
| ------------------------- | ------------ |
| last\_detection\_position | 最近一次检测到的位置坐标 |
| last\_number\_value       | 最近一次识别的数字值   |

**位置变量参数**: 多个检测节点支持自定义位置保存的变量名（position\_key参数），默认为 `last_detection_position`，可用于多检测点场景。

***

## 节点状态

| 状态  | 说明              |
| --- | --------------- |
| 成功  | 节点执行成功          |
| 失败  | 节点执行失败          |
| 运行中 | 节点正在执行，需要继续tick |

***

## 常见用途速查

| 用途      | 推荐节点组合                 |
| ------- | ---------------------- |
| 点击按钮    | 图像匹配节点 → 鼠标点击节点        |
| 检测文字并点击 | OCR检测节点 → 鼠标点击节点       |
| 循环执行    | 顺序节点 + 变量判断节点 + 设置变量节点 |
| 多方案尝试   | 选择节点 + 多个条件节点          |
| 并行检测    | 并行节点 + 多个条件节点          |
| 等待加载    | 延时节点                   |
| 数值判断    | 数字比较节点                 |
| 状态控制    | 变量判断节点 + 设置变量节点        |
