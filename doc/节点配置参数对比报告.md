## 鼠标节点实现差异详细分析

### MouseScrollNode (鼠标滚动节点)

#### 旧项目实现

```python
# 旧项目参数
distance = self.config.get("distance", 5)      # 滚动距离
clicks = self.config.get("clicks", 1)          # 滚动次数
direction = self.config.get("direction", "向上") # 滚动方向

# 旧项目逻辑
if direction == "向上":
    scroll_distance = abs(distance)
    scroll_direction = "垂直"
elif direction == "向下":
    scroll_distance = -abs(distance)
    scroll_direction = "垂直"
elif direction == "向左":
    scroll_distance = -abs(distance)
    scroll_direction = "水平"
elif direction == "向右":
    scroll_distance = abs(distance)
    scroll_direction = "水平"

context.execute_mouse_scroll(scroll_distance, clicks, scroll_direction)
```

**特点**:
- 支持四个方向滚动（上下左右）
- 区分垂直和水平滚动
- 使用 `distance` + `clicks` + `direction` 组合参数
- 不支持指定滚动位置

#### 新项目实现

```python
# 新项目参数
amount = self.config.get_int("amount", 1)           # 滚动量（正负表示方向）
position = self.config.get("position", None)        # 滚动位置
use_blackboard = self.config.get_bool("use_blackboard", False)
position_key = self.config.get("position_key", "last_detection_position")

# 新项目逻辑
scroll_position = self.position
if self.use_blackboard:
    bb_position = context.blackboard.get(self.position_key)
    if bb_position:
        scroll_position = bb_position

if scroll_position:
    context.execute_mouse_move(scroll_position, relative=False)

context.execute_mouse_scroll(self.amount, scroll_position)
```

**特点**:
- 使用 `amount` 的正负值表示方向（正数向上/右，负数向下/左）
- 支持指定滚动位置（可先移动鼠标到指定位置再滚动）
- 支持从黑板获取位置（与其他检测节点联动）
- 不再区分垂直/水平滚动，由底层根据 amount 值自动判断

#### 差异对比

| 特性 | 旧项目 | 新项目 |
|------|--------|--------|
| 方向控制 | `direction` 参数（上下左右） | `amount` 正负值 |
| 滚动位置 | 不支持 | 支持 `position` 参数 |
| 黑板联动 | 不支持 | 支持 `use_blackboard` |
| 水平滚动 | 显式支持 | 通过 amount 负值支持 |
| 参数数量 | 3个 | 4个 |

#### 迁移影响

**严重**: 旧项目的行为树JSON无法直接迁移，需要完全重新配置滚动节点。

#### 新项目实现

```python
relative = self.config.get_bool("relative", False)
use_blackboard = self.config.get_bool("use_blackboard", False)
position_key = self.config.get("position_key", ...)
move_type = self.config.get("move_type", "move")
drag_button = self.config.get("drag_button", "left")
end_position = self.config.get("end_position", None)
use_blackboard_end = self.config.get_bool("use_blackboard_end", False)
position_key_end = self.config.get("position_key_end", ...)
drag_duration = self.config.get_int("drag_duration", 0)

# 新项目拖拽逻辑（非阻塞模式）
def _non_blocking_drag(self, context, start_pos):
    # 首次调用时按下鼠标
    if not self._drag_started:
        self._drag_started = True
        self._drag_step = 0
        context.execute_mouse_click(self.drag_button, start_pos, "down", 0)
    
    # 分步移动（非阻塞）
    if self.drag_duration > 0:
        steps = max(10, self.drag_duration // 20)
        if self._drag_step < steps:
            # ... 计算当前位置并移动 ...
            self._drag_step += 1
            return NodeStatus.RUNNING  # 返回运行状态，下次继续
    else:
        # 无动画直接移动
        if self._drag_step == 0:
            context.execute_mouse_move(end_pos, relative=False)
            self._drag_step = 1
    
    # 释放鼠标
    context.execute_mouse_click(self.drag_button, end_pos, "up", 0)
    return NodeStatus.SUCCESS
```

**特点**:
- ✅ 已实现 `smooth` 参数（使用 InputController 的平滑移动功能）
- 拖拽使用阻塞模式（与旧项目一致）
- 拖拽步数计算公式：`drag_duration // 50`（与旧项目一致）

## 条件节点被移除参数分析

### 1. ColorConditionNode - `min_pixels` 参数

#### 作用
```python
# 旧项目代码
min_pixels = self.config.get("min_pixels", 1)
matched, position, match_pixels = ColorRecognizer.match_color(...)

if matched and match_pixels >= min_pixels:
    # 匹配成功
```

**功能**: 设置颜色匹配的最小像素数量阈值。只有当匹配到的像素数量 >= min_pixels 时，才判定为匹配成功。

#### 移除影响

| 场景 | 旧项目行为 | 新项目行为 |
|------|------------|------------|
| 匹配到1个像素 | 可配置是否成功 | 直接成功 |
| 匹配到多个像素 | 需要达到阈值 | 直接成功 |

**影响程度**: ⚠️ 中等

- **问题场景**: 如果屏幕上有噪点或误匹配，新项目可能会误判为成功
- **建议**: 如果需要精确控制匹配像素数量，需要在代码层面恢复此参数

### 3. OCRConditionNode - `match_mode` 参数

#### 作用
```python
# 旧项目代码
match_mode = self.config.get("match_mode", "any")

keyword_list = [k.strip().lower() for k in keywords.split(",")]
text_lower = text.lower()

if match_mode == "any":
    matched = any(kw in text_lower for kw in keyword_list)
elif match_mode == "all":
    matched = all(kw in text_lower for kw in keyword_list)
```

**功能**: 控制关键词匹配模式：
- `any`: 任意一个关键词匹配即成功
- `all`: 所有关键词都必须匹配

#### 移除影响

| 场景 | 旧项目行为 | 新项目行为 |
|------|------------|------------|
| 多关键词匹配 | 可选择 any/all | 固定为 any |

**影响程度**: ⚠️ 中等

- **问题场景**: 如果需要检测多个关键词同时存在，新项目无法实现
- **建议**: 如果需要 `all` 模式，需要恢复此参数
