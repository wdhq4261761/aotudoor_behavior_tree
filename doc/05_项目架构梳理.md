## 1. 项目总览

### 1.1 项目定位

AutoDoor 行为树系统是一个**独立的可视化行为树编辑与执行框架**，面向 Windows 平台的自动化场景：

| 应用领域 | 典型场景 |
|---------|---------|
| 游戏辅助 | 自动任务、自动战斗、自动采集 |
| RPA 流程 | 自动化办公、数据录入、报表生成 |
| 测试自动化 | UI测试、回归测试、压力测试 |

### 4.3 节点核心接口

```python
    node_id: str                                # 唯一标识

    children: List[Node]                        # 子节点列表
    parent: Optional[Node]                      # 父节点引用
    
    def tick(self, context: ExecutionContext) -> NodeStatus:
        """执行节点逻辑，返回执行状态"""
        pass
    
    def reset(self, reset_counters: bool = True) -> None:
        """重置节点状态"""
        pass
    
    def abort(self, context: ExecutionContext) -> None:
        """中止节点执行"""
        pass
    
    def to_dict(self) -> Dict[str, Any]:
        """序列化为字典"""
        pass
    
    @classmethod
    def from_dict(cls, data: Dict[str, Any]) -> "Node":
        """从字典反序列化"""
        pass
```

```
│                              ▼                             │  │
```

### 5.2 引擎核心特性

| 特性 | 说明 |
|-----|------|
| 执行模式 | 独立守护线程循环 tick |
| Tick 间隔 | 默认 50ms，可配置 |
| 状态回调 | on_status_change / on_node_status 通知外部 |
| 线程安全 | threading.Lock 保护启动/停止操作 |

## 13. 设计决策与权衡

| 决策 | 选择 | 原因 |
|-----|------|------|
| GUI 框架 | CustomTkinter | 原项目使用，暗色主题原生支持 |
| 画布实现 | Tkinter Canvas | 轻量级，无需额外依赖 |
| 节点通信 | 黑板模式 | 解耦节点间依赖，支持观察者通知 |
| 撤销系统 | Command 模式 | 标准方案，支持细粒度操作回退 |
| 输入抽象 | 延迟初始化 | 避免启动时加载不必要的驱动 |
| 序列化 | 扁平化节点+连线 | 兼容编辑器与引擎两种数据格式 |
| 打包 | PyInstaller | Windows 单文件分发，支持 DLL 打包 |
| 资源路径管理 | ResourceManager 单例 | 统一管理开发/打包环境的资源路径 |
| Tesseract 路径 | 自动检测 | 简化用户配置，支持打包环境 |
| 崩溃恢复 | 静默自动恢复 | 提升用户体验，避免重复确认 |

## 14. 关键技术要点

### 14.1 节点重置机制

| 场景 | 方法 | 行为 |
|-----|------|------|
| 完全重置 | reset(reset_counters=True) | 重置所有状态和计数器 |
| 重试重置 | _reset_for_retry() | 设置 status=RUNNING，保留重试计数器 |
| 重复重置 | _reset_for_repeat() | 调用 reset(False)，保留重复计数器 |

### 14.2 焦点保护机制

删除快捷键（Delete/Backspace）在输入框获得焦点时不触发节点删除：

```python
def _on_delete_key(self, event):
    focused = self.winfo_toplevel().focus_get()
    if focused:
        widget_type = str(type(focused).__name__)
        if widget_type in ("CTkEntry", "Entry", "CTkTextbox", "Text"):
            return None  # 允许输入框正常处理删除
    self._delete_selected()
    return "break"
```

### 14.3 自动保存与崩溃恢复

| 清理类型 | 触发时机 | 说明 |
|---------|---------|------|
| 崩溃文件清理 | 应用启动时 | 清理超过7天的崩溃恢复文件 |
| 自动保存清理 | 手动保存成功后 | 清理所有自动保存文件 |
| 备份轮转 | 每次自动保存时 | 保留最近3个自动保存备份 |
