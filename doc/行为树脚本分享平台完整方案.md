## 一、项目概述

### 1.1 项目背景

您已拥有一套完整的行为树自动化工具，具备可视化节点编辑、多种节点类型、装饰器参数配置、黑板数据共享、多格式序列化等核心能力。本方案旨在为现有用户群体搭建一个用于讨论交流、分享自动化脚本的社区平台。

### 1.2 核心需求

| 需求 | 描述 | 优先级 |
|------|------|--------|
| 付费内容 | 创作者设置免费/付费文章，用户付款解锁 | 最高 |
| 订阅制 | 月度/年度订阅，解锁创作者全部付费内容 | 最高 |
| 脚本上传 | 创作者上传脚本文件（JSON/YAML/TEXT） | 最高 |
| 脚本下载 | 用户购买后下载加密脚本文件 | 最高 |
| 支付接入 | 仅需支付宝 + 微信支付 | 最高 |
| 社区讨论 | 帖子、回复、评论、标签、搜索 | 高 |
| 内容审核 | 审核工作台 + 机器自动审核 | 高 |
| 用户系统 | 注册、登录、个人主页 | 中 |

### 1.3 约束条件

- **支付渠道**：仅支付宝 + 微信支付，不涉及 Stripe 等国际支付
- **无分成系统**：平台不涉及多作者分成结算（创作者自行通过付费内容获得收入）
- **国内运营**：需符合国内法律法规和备案要求
- **低成本启动**：优先选择成熟插件，减少定制开发量

## 二、商业模式设计

### 2.1 订阅制模式说明

平台采用**创作者订阅制**模式，参考知识星球、小报童等国内成熟平台的运营经验：

```
创作者开设付费专栏 → 用户订阅/单篇购买 → 支付宝/微信付款 → 解锁付费内容
```

**核心特征**：

- 平台仅提供技术服务，不直接处理创作者收益分配
- 每位创作者独立设置付费内容价格和订阅方案
- 用户付款直接进入平台账户，创作者收益通过平台结算
- 平台通过收取技术服务费（如交易手续费）盈利

### 2.2 收入来源

| 收入来源 | 说明 | 参考费率 |
|---------|------|---------|
| 交易手续费 | 每笔付费内容交易抽取一定比例 | 5%-15% |
| 增值服务 | 创作者高级功能（数据分析、自定义域名等） | 按月/年收费 |
| 广告收入 | 平台展示广告（后期） | 按展示/点击计费 |

### 2.3 用户角色

| 角色 | 权限 | 说明 |
|------|------|------|
| 普通用户 | 浏览免费内容、购买付费内容、评论 | 平台基础用户 |
| 创作者 | 发布免费/付费内容、上传脚本、管理专栏 | 需申请或自动升级 |
| 管理员 | 全站管理、内容审核、用户管理、财务统计 | 平台运营团队 |

## 三、法律合规分析

### 3.1 合规风险评估

| 风险领域 | 风险等级 | 说明 |
|---------|---------|------|
| 虚拟货币/非法金融 | 🟢 极低 | 无积分发行、无虚拟货币，不涉及 |
| 支付牌照要求 | 🟢 极低 | 通过第三方支付插件接入，不直接处理资金清算 |
| 反洗钱（AML） | 🟢 极低 | 无提现功能，无资金池 |
| 内容合规 | 🟡 中高 | 自动化脚本分发的法律边界需严格管控 |
| 平台收费合规 | 🟡 中 | 需符合《网络交易平台收费行为合规指南》 |
| 消费者权益保护 | 🟡 中 | 付费内容涉及退款、质量保障等要求 |
| 税务合规 | 🟡 中 | 平台收入需依法纳税 |
| 版权风险 | 🟡 中 | 脚本知识产权归属需明确 |

### 3.2 自动化脚本分发的法律边界

自动化脚本本身并非非法，但其用途决定了法律风险：

| 场景分类 | 法律风险 | 风险等级 | 平台应对措施 |
|---------|---------|---------|------------|
| 合规自动化（办公效率、数据处理） | 低 | 🟢 绿色 | 允许并鼓励 |
| 游戏辅助/外挂类脚本 | 高 | 🔴 红色 | **严格禁止**，建立关键词过滤 |
| 批量刷单、抢票、抓包等扰乱市场行为 | 高 | 🔴 红色 | **严格禁止** |
| 爬取他人数据的脚本 | 中高 | 🟠 橙色 | 限制分享，要求声明数据来源 |
| 绕过反爬、验证码的脚本 | 高 | 🔴 红色 | **严格禁止** |
| 平台作为分发方的责任 | 中高 | 🟠 橙色 | 建立免责条款 + 内容审核 |

### 3.3 关键法律法规要点

1. **《电子商务法》**：平台作为电子商务经营者，需履行经营者义务
2. **《网络安全法》**：建立内容审核制度和用户信息保护机制
3. **《个人信息保护法》**：处理用户个人信息需遵循最小化、知情同意原则
4. **《消费者权益保护法》**：付费内容需提供合理的退款机制
5. **《网络交易平台收费行为合规指南》**（2025年7月施行）：收费透明、禁止不合理收费
6. **《著作权法》**：明确脚本知识产权归属，建立侵权投诉机制

## 四、国内备案规范

### 4.1 必要资质清单

| 资质/要求 | 是否必要 | 说明 |
|----------|---------|------|
| 企业营业执照 | **必要** | 经营性平台必须以企业主体运营 |
| 经营性ICP许可证 | **必要** | 涉及付费服务，必须办理 |
| 域名实名认证 | **必要** | 备案前必须完成，信息与企业一致 |
| 公安备案 | **必要** | ICP备案后30日内完成 |
| EDI许可证 | 建议办理 | 涉及在线交易处理，建议申请 |
| SSL证书（HTTPS） | **必要** | 全站HTTPS加密 |
| 网络文化经营许可证 | 视情况 | 如涉及文化内容分发可能需要 |

### 4.2 备案流程与时间线

```
第1-2周：注册企业（含互联网信息服务范围）
    ↓
第2周：购买域名 + 完成实名认证
    ↓
第2周：租用大陆云服务器（阿里云/腾讯云）
    ↓
第3-5周：提交ICP备案（通过云服务商代办）
    ↓
第6-10周：申请经营性ICP许可证
    ↓
第10-18周：申请EDI许可证（建议）
    ↓
ICP备案后30日内：公安备案
```

> **注意**：ICP备案和许可证申请期间，可以先在本地或海外服务器进行开发测试，备案完成后再切换到国内服务器正式上线。

## 七、核心功能实现方案

### 7.1 付费内容系统

**实现方式**：Paid Memberships Pro + 内容访问控制

```
创作者发布文章 → 设置为"付费"（关联会员等级） → 用户浏览时检测权限
    → 未购买：显示付费提示 + 购买按钮
    → 已购买：显示完整内容 + 脚本下载链接
```

**具体配置**：

1. 创建会员等级（如"免费用户"、"专栏订阅者"）
2. 创作者发布文章时，在编辑器中设置内容访问级别
3. 付费内容区域使用 `[pmpro_member]` 短代码包裹
4. 非会员用户看到付费提示和购买选项

### 7.2 订阅制实现

**实现方式**：Paid Memberships Pro 会员等级 + 循环付款

```
用户选择订阅方案 → 选择支付宝/微信支付 → 完成付款 → 自动升级会员等级
    → 可访问该创作者全部付费内容
    → 到期前自动提醒续费
    → 到期后自动降级为免费用户
```

**订阅方案设计**：

| 方案 | 价格 | 权限 | 说明 |
|------|------|------|------|
| 免费用户 | ¥0 | 浏览免费内容、评论 | 基础权限 |
| 月度订阅 | ¥19.9/月 | 解锁全部付费内容 + 脚本下载 | 适合体验用户 |
| 季度订阅 | ¥49.9/季 | 同上 | 约8.3折 |
| 年度订阅 | ¥149.9/年 | 同上 + 优先客服 | 约6.3折，鼓励年付 |

### 7.3 脚本上传下载

**实现方式**：Download Monitor + 阿里云OSS

**上传流程**：

```
创作者在文章编辑页 → 点击"添加下载" → 上传脚本文件（JSON/YAML/TEXT）
    → 文件存储到阿里云OSS（加密）
    → 生成下载短码嵌入文章
    → 下载权限与文章付费状态绑定
```

**下载流程**：

```
用户打开付费文章 → 已购买：显示下载按钮 → 点击下载 → 验证权限 + 记录下载次数
    → 从OSS获取文件 → 返回给用户
    未购买：显示"购买后解锁下载"
```

### 7.4 支付接入

- 插件：Paid Memberships Pro + 支付宝扩展

## 八、脚本加密分享机制

### 8.1 多层次保护策略

| 保护层级 | 技术方案 | 说明 |
|---------|---------|------|
| 传输层加密 | HTTPS + OSS私有签名 | 防止传输过程中被截获 |
| 存储层加密 | AES-256加密存储 | 服务端加密存储脚本文件 |
| 访问控制 | 会员权限 + 下载次数限制 | 限制未授权访问 |
| 设备绑定 | 可选：绑定下载设备数 | 防止大规模传播 |
| 水印标识 | 脚本内嵌入唯一标识 | 追踪泄露源头 |

### 8.2 实现方案

**服务端加密存储**：

```php
// 上传时加密
function encrypt_script($file_content, $key) {
    $iv = openssl_random_pseudo_bytes(16);
    $encrypted = openssl_encrypt(
        $file_content, 'AES-256-CBC', $key, OPENSSL_RAW_DATA, $iv
    );
    return base64_encode($iv . $encrypted);
}

// 下载时解密
function decrypt_script($encrypted_content, $key) {
    $data = base64_decode($encrypted_content);
    $iv = substr($data, 0, 16);
    $encrypted = substr($data, 16);
    return openssl_decrypt(
        $encrypted, 'AES-256-CBC', $key, OPENSSL_RAW_DATA, $iv
    );
}
```

**OSS私有签名URL**：

```php
// 生成带时效的签名下载URL（有效期30分钟）
function generate_download_url($oss_key, $user_id) {
    // 验证用户权限
    if (!user_has_access($user_id, $oss_key)) {
        return null;
    }
    // 记录下载
    record_download($user_id, $oss_key);
    // 生成签名URL
    return $oss_client->signUrl($oss_key, 1800);
}
```

## 九、内容审核体系

### 9.1 多层审核机制

| 审核层级 | 实现方式 | 审核内容 |
|---------|---------|---------|
| 机器自动审核 | 敏感词过滤 + 文件类型检测 | 违禁关键词、恶意代码、危险文件 |
| 人工审核 | 审核工作台 + 审核人员 | 脚本功能审核、帖子内容审核 |
| 用户举报 | 举报按钮 + 处理流程 | 用户发现违规内容后举报 |
| 后台监控 | 日志审计 + 异常行为检测 | 批量上传、异常下载、账号异常 |

### 9.2 审核流程

```
创作者上传脚本/发布帖子
    ↓
机器自动审核（敏感词、文件类型、代码扫描）
    ↓
通过 → 直接发布
拒绝 → 标记为"待审核" + 通知创作者
    ↓
人工审核（付费内容必须人工审核）
    ↓
通过 → 发布上线

## 十三、风险缓解措施

### 13.1 内容合规风险

| 风险 | 缓解措施 |
|------|---------|
| 违规脚本发布 | 多层审核机制 + 违禁关键词库 + 用户举报 |
| 版权纠纷 | 建立DMCA投诉流程 + 创作者原创声明 |
| 平面连带责任 | 用户协议明确免责条款 + 及时处理违规内容 |

### 13.2 技术安全风险

| 风险 | 缓解措施 |
|------|---------|
| 网站被攻击 | Wordfence防火墙 + 定期安全扫描 + 自动备份 |
| 数据泄露 | HTTPS加密 + 数据库加密 + 最小权限原则 |
| 服务中断 | 自动备份 + 云服务器监控 + 快速恢复方案 |
| 脚本被破解 | AES-256加密 + 设备绑定 + 下载次数限制 + 水印追踪 |

### 13.3 运营风险

| 风险 | 缓解措施 |
|------|---------|
| 用户增长缓慢 | 冷启动策略 + 创作者激励 + SEO优化 |
| 付费转化率低 | 免费试用 + 合理定价 + 优质内容吸引 |
| 创作者流失 | 收益分成优化 + 流量扶持 + 社区荣誉体系 |

## 十四、总结

### 核心结论

1. **WordPress + 插件方案**是当前条件下最优选择：国内支付生态最完善、开发成本最低、上线速度最快
2. **订阅制模式**大幅降低了金融合规风险，参考知识星球、小报童等成熟平台的运营经验
3. **前期投入极低**（0.3-2万元），适合快速验证市场和最小化启动风险
4. **内容合规**是唯一需要持续关注的核心风险，需建立完善的审核体系
5. **可渐进式发展**：从免费插件组合启动，根据运营数据逐步升级付费插件和定制功能

    ↓

```
